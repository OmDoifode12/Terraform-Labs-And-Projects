# Day 34 – Copy Data to S3 Using Terraform

---

## Task Overview

As part of AWS data migration activities, the Nautilus DevOps team needs to upload local data files into an existing Amazon S3 bucket using Terraform.

For this task:

* Existing S3 Bucket:

```text id="cds301"

```


```text id="cds312"

```

---

## Step 4: Add S3 Object Resource

Add the following block below the existing S3 bucket resource:

```hcl id="cds313"
resource "aws_s3_object" "upload_file" {
  bucket = aws_s3_bucket.my_bucket.bucket
  key    = "datacenter.txt"
  source = "/tmp/datacenter.txt"
}
```

---

# Complete Terraform Configuration

```hcl id="cds314"
resource "aws_s3_bucket" "my_bucket" {
  bucket = "datacenter-cp-21736"
  acl    = "private"

  tags = {
    Name = "datacenter-cp-21736"
  }
}

resource "aws_s3_object" "upload_file" {
  bucket = aws_s3_bucket.my_bucket.bucket
  key    = "datacenter.txt"
  source = "/tmp/datacenter.txt"
}
```

---

# Code Explanation

| Block         | Purpose            |
| ------------- | ------------------ |
| aws_s3_bucket | Manages S3 bucket  |
| aws_s3_object | Uploads file to S3 |
| bucket        | Target S3 bucket   |
| key           | Object name in S3  |
| source        | Local file path    |

# Best Practices

* Use versioning for important S3 buckets
* Use encryption for sensitive files
* Organize objects using prefixes/folders
* Avoid public access unless necessary
* Monitor S3 storage costs

---

# Key Learnings

* Terraform can upload files into S3
* S3 stores files as objects
* aws_s3_object manages S3 uploads
* Infrastructure as Code simplifies automation
* Terraform tracks uploaded objects in state
