# Day 14 – Create IAM User Using Terraform

---

## Task Overview

Identity and Access Management (IAM) is one of the foundational AWS services used to securely manage access to cloud resources. As part of their cloud migration strategy, the Nautilus DevOps team is configuring IAM resources using Terraform to automate user management.

For this task, create an IAM user named `iamuser_rose` using Terraform.

---

# Objective

| Property          | Value          |
| ----------------- | -------------- |
| Resource Type     | IAM User       |
| User Name         | `iamuser_rose` |
| Service           | AWS IAM        |
| Provisioning Tool | Terraform      |

---

# Understanding IAM Users

An IAM User represents a person or application that interacts with AWS resources.

IAM Users can have:

* Login credentials
* Access keys
* Permissions
* Policies
* Group memberships

Common Use Cases:

* Developers
* DevOps Engineers
* CI/CD Pipelines
* Automation Scripts

# Step-by-Step Implementation

## Step 1: Navigate to Terraform Directory

```bash
cd /home/bob/terraform
```

### Explanation

This is the Terraform working directory where infrastructure configuration files are stored.

---

## Step 2: Create main.tf

```bash
touch main.tf
```

### Explanation

Terraform resources are defined inside `.tf` files.

---

## Step 3: Add Terraform Configuration

```hcl
resource "aws_iam_user" "iamuser_rose" {
  name = "iamuser_rose"
}
```

### Explanation

| Parameter    | Description                |
| ------------ | -------------------------- |
| aws_iam_user | Creates an IAM user        |
| name         | Specifies the IAM username |

---

## Step 4: Initialize Terraform

```bash
terraform init
```

### Explanation

Downloads required providers and initializes Terraform.

---

## Step 5: Validate Configuration

```bash
terraform validate
```

### Explanation

Checks Terraform syntax and configuration correctness.

---

## Step 6: Preview Changes

```bash
terraform plan
```

### Explanation

Displays resources Terraform will create before deployment.

---

## Step 7: Apply Configuration

```bash
terraform apply -auto-approve
```

### Explanation

Creates the IAM user in AWS.

---

## Step 8: Verify IAM User

```bash
aws iam list-users
```

### Explanation

Lists IAM users and verifies successful creation.

---

## Step 9: Verify Specific User

```bash
aws iam get-user --user-name iamuser_rose
```

### Explanation

Displays detailed information about the IAM user.

# Best Practices

* Follow the Principle of Least Privilege.
* Use IAM Groups for permission management.
* Rotate credentials regularly.
* Enable MFA whenever possible.
* Avoid using root account credentials.

---

# Key Learnings

* IAM manages authentication and authorization in AWS.
* Terraform can automate IAM resource creation.
* IAM Users represent individuals or applications.
* Access management is critical for cloud security.

# Interview Questions & Answers

### 1. What is AWS IAM?

AWS Identity and Access Management (IAM) is a service used to securely manage access to AWS resources.

---

### 2. What is an IAM User?

An IAM User is an identity created within AWS for a person or application.

---

### 3. Which Terraform resource creates an IAM User?

```hcl
aws_iam_user
```

---

### 4. What is the Principle of Least Privilege?

Users should receive only the permissions required to perform their tasks.

---

### 5. What is the difference between IAM User and IAM Role?

| IAM User            | IAM Role                       |
| ------------------- | ------------------------------ |
| Permanent identity  | Temporary identity             |
| Has credentials     | Assumed when needed            |
| Used by people/apps | Used by services and workloads |

---

### 6. Why should MFA be enabled?

To add an additional layer of security beyond passwords.
