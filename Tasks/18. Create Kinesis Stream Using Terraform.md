# Day 18 – Create Kinesis Stream Using Terraform

---

## Task Overview

As part of the AWS cloud migration and real-time analytics initiative, the Nautilus DevOps team needs to provision a Kinesis Data Stream using Terraform. Amazon Kinesis enables real-time ingestion, processing, and analysis of streaming data at massive scale.

For this task, create a Kinesis stream named `nautilus-stream` using Terraform.

# Understanding Amazon Kinesis

Amazon Kinesis is a real-time data streaming service used to collect, process, and analyze large volumes of streaming data.

Common Use Cases:

* Real-time analytics
* Log processing
* Application monitoring
* IoT telemetry
* Financial transactions
* Streaming dashboards

---

# Understanding Kinesis Streams

A Kinesis Data Stream continuously captures and stores streaming data records.

Example Flow:

```text id="jlwmc1"
Applications → Kinesis Stream → Analytics / Consumers
```

---

# Understanding Shards

A shard is the base throughput unit of a Kinesis stream.

Each shard provides:

| Operation | Capacity |
| --------- | -------- |
| Write     | 1 MB/sec |
| Read      | 2 MB/sec |

For this task:

```text id="jlwmc2"
shard_count = 1
```

is sufficient.

---

# Understanding Stream Modes

Kinesis supports two stream modes:

| Mode        | Description             |
| ----------- | ----------------------- |
| PROVISIONED | Manual shard management |
| ON_DEMAND   | Automatic scaling       |

For this task:

```text id="jlwmc3"
PROVISIONED
```

mode is used.

---

## Step 3: Open main.tf

```bash id="jlwmc6"
vi main.tf
```

Press:

```text id="jlwmc7"
i
```

to enter INSERT mode.

---

## Step 4: Add Terraform Configuration

```hcl id="jlwmc8"
resource "aws_kinesis_stream" "nautilus_stream" {
  name             = "nautilus-stream"
  shard_count      = 1
  retention_period = 24

  stream_mode_details {
    stream_mode = "PROVISIONED"
  }
}
```

---

# Code Explanation

| Block               | Purpose                 |
| ------------------- | ----------------------- |
| aws_kinesis_stream  | Creates Kinesis stream  |
| name                | Stream name             |
| shard_count         | Number of shards        |
| retention_period    | Data retention duration |
| stream_mode_details | Defines stream mode     |
| PROVISIONED         | Manual shard management |
