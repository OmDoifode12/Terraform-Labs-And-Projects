# Day 26 – Attach Elastic IP Using Terraform

---

## Task Overview

As part of AWS infrastructure automation, the Nautilus DevOps team needs to associate an existing Elastic IP (EIP) with an EC2 instance using Terraform. Elastic IP addresses provide static public IP addresses for AWS resources.

For this task:

* EC2 Instance Name: `devops-ec2`
* Elastic IP Name: `devops-ec2-eip`

Attach the Elastic IP to the EC2 instance using Terraform.

---

# Objective

| Property          | Value            |
| ----------------- | ---------------- |
| Resource Type     | Elastic IP       |
| EC2 Instance      | `devops-ec2`     |
| Elastic IP        | `devops-ec2-eip` |
| AWS Region        | `us-east-1`      |
| Provisioning Tool | Terraform        |

---

# Understanding Elastic IP

An Elastic IP (EIP) is a static public IPv4 address provided by AWS.

Benefits:

* Static public access
* Persistent IP address
* Failover support
* Reliable external connectivity

---

# Why Attach an Elastic IP?

Normally, EC2 public IPs can change after restart.

Elastic IP solves this problem:

```text id="eip01"
Elastic IP = Permanent Public IP
```

---

# Understanding EIP Association

An Elastic IP must be associated with:

* EC2 instance
* Network interface

Terraform handles this association automatically.

---

# Architecture Overview

```text id="eip02"
Elastic IP → Attached → EC2 Instance
```

## Step 3: Update Elastic IP Resource

Modify the EIP resource:

```hcl id="eip08"
resource "aws_eip" "ec2_eip" {
  instance = aws_instance.ec2.id

  tags = {
    Name = "devops-ec2-eip"
  }
}
```

---

# Complete Terraform Configuration

```hcl id="eip09"
resource "aws_instance" "ec2" {
  ami           = "ami-0c101f26f147fa7fd"
  instance_type = "t2.micro"
  subnet_id     = "subnet-f0bab45f0e8f1f059"

  vpc_security_group_ids = [
    "sg-5ce74894a50df9d9b"
  ]

  tags = {
    Name = "devops-ec2"
  }
}

resource "aws_eip" "ec2_eip" {
  instance = aws_instance.ec2.id

  tags = {
    Name = "devops-ec2-eip"
  }
}
```

---

# Code Explanation

| Block        | Purpose                      |
| ------------ | ---------------------------- |
| aws_instance | Creates EC2 instance         |
| aws_eip      | Creates Elastic IP           |
| instance     | Attaches EIP to EC2 instance |
| tags         | Adds resource tags           |
