# Day 04 – Create VPC with CIDR Using Terraform

---

## Task Overview

As part of the AWS cloud migration initiative, the Nautilus DevOps team is provisioning networking infrastructure using Terraform. A Virtual Private Cloud (VPC) provides an isolated networking environment where AWS resources can securely communicate with each other.

For this task, create a VPC named `devops-vpc` with the IPv4 CIDR block `192.168.0.0/24` in the `us-east-1` region using Terraform.

---

# Objective

| Property          | Value            |
| ----------------- | ---------------- |
| Resource Type     | VPC              |
| VPC Name          | `devops-vpc`     |
| CIDR Block        | `192.168.0.0/24` |
| Region            | `us-east-1`      |
| Provisioning Tool | Terraform        |

---

# Understanding CIDR

CIDR (Classless Inter-Domain Routing) defines the IP address range available within a network.

Example:

```text
192.168.0.0/24
```

* Network Address: `192.168.0.0`
* Available IP Addresses: `256`
* Usable Host Addresses: `254`

# Step-by-Step Implementation

## Step 1: Navigate to Terraform Directory

```bash
cd /home/bob/terraform
```

### Explanation

This is the Terraform working directory where configuration files are stored.

---

## Step 2: Create main.tf

```bash
touch main.tf
```

### Explanation

Terraform configuration files use the `.tf` extension.

---

## Step 3: Add Terraform Configuration

```hcl
provider "aws" {
  region = "us-east-1"
}

resource "aws_vpc" "devops_vpc" {
  cidr_block = "192.168.0.0/24"

  tags = {
    Name = "devops-vpc"
  }
}
```

### Explanation

| Parameter    | Description               |
| ------------ | ------------------------- |
| provider aws | Configures AWS provider   |
| region       | AWS deployment region     |
| aws_vpc      | Creates a VPC             |
| cidr_block   | Defines IP address range  |
| tags         | Assigns a name to the VPC |

---

## Step 4: Initialize Terraform

```bash
terraform init
```

### Explanation

Downloads required providers and initializes the Terraform project.

---

## Step 5: Validate Configuration

```bash
terraform validate
```

### Explanation

Checks the configuration syntax and validates the code.

---

## Step 6: Preview Infrastructure Changes

```bash
terraform plan
```

### Explanation

Displays resources Terraform will create without making actual changes.

---

## Step 7: Apply Configuration

```bash
terraform apply -auto-approve
```

### Explanation

Creates the VPC in AWS automatically.

---

## Step 8: Verify VPC

```bash
aws ec2 describe-vpcs --region us-east-1
```

### Explanation

Lists available VPCs and confirms successful creation.

### 1. What is a VPC?

A Virtual Private Cloud (VPC) is a logically isolated network in AWS where resources can be deployed securely.

---

### 2. What is CIDR?

CIDR (Classless Inter-Domain Routing) defines a range of IP addresses for a network.

---

### 3. What does `192.168.0.0/24` mean?

It represents a network with 256 IP addresses, of which 254 are usable.
