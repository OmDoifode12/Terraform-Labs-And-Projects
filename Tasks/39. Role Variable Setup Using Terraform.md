# Day 39 – Role Variable Setup Using Terraform

---

## Task Overview

As part of AWS Identity and Access Management (IAM) automation, the Nautilus DevOps team needs to create an IAM Role using Terraform while implementing variables for reusable and modular configurations.

For this task:

* Create an AWS IAM Role.
* Store the IAM Role name:

```text id="rvs01"
iamrole_ravi
```

inside a Terraform variable named:

```text id="rvs02"
KKE_iamrole
```

---

# Objective

| Property          | Value          |
| ----------------- | -------------- |
| Resource Type     | IAM Role       |
| IAM Role Name     | `iamrole_ravi` |
| Variable Name     | `KKE_iamrole`  |
| AWS Service       | IAM            |
| AWS Region        | `us-east-1`    |
| Provisioning Tool | Terraform      |

---

# Understanding IAM Roles

An IAM Role is an AWS identity with temporary permissions that can be assumed by trusted AWS services or users.

Common Use Cases:

* EC2 Instance Roles
* Lambda Execution Roles
* Cross-Account Access
* Temporary Security Access

---

# Why Use Variables in Terraform?

Terraform variables improve:

| Benefit     | Description                   |
| ----------- | ----------------------------- |
| Reusability | Reuse infrastructure easily   |
| Flexibility | Dynamic configuration values  |
| Modularity  | Cleaner Terraform code        |
| Scalability | Easier environment management |

Instead of hardcoding:

```hcl id="rvs03"
name = "iamrole_ravi"
```

we use:

```hcl id="rvs04"
name = var.KKE_iamrole
```

# Configuring main.tf

```bash id="rvs13"
vi main.tf
```

Press:

```text id="rvs14"
i
```

to enter INSERT mode.

---

## Step 7: Add Terraform Configuration

```hcl id="rvs15"
resource "aws_iam_role" "iam_role" {
  name = var.KKE_iamrole

  assume_role_policy = jsonencode({
    Version = "2012-10-17"

    Statement = [
      {
        Effect = "Allow"

        Principal = {
          Service = "ec2.amazonaws.com"
        }

        Action = "sts:AssumeRole"
      }
    ]
  })

  tags = {
    Name = var.KKE_iamrole
  }
}
```

---

# Code Explanation

| Block              | Purpose                        |
| ------------------ | ------------------------------ |
| aws_iam_role       | Creates IAM Role               |
| name               | IAM Role name                  |
| assume_role_policy | Defines trusted entity         |
| jsonencode()       | Converts Terraform map to JSON |
| tags               | Adds AWS resource tags         |
| var.KKE_iamrole    | References Terraform variable  |

---

# Understanding assume_role_policy

IAM Roles require a:

```text id="rvs16"
Trust Policy
```

which defines:

```text id="rvs17"
Who can assume the role
```

In this example:

```text id="rvs18"
EC2 instances can assume the role
```

---

# Understanding jsonencode()

```hcl id="rvs19"
jsonencode()
```

converts Terraform data structures into valid JSON format required by AWS IAM.

---

# Best Practices

* Use variables for reusable values
* Avoid hardcoding IAM role names
* Apply least-privilege permissions
* Use descriptive IAM trust policies
* Organize Terraform files clearly
