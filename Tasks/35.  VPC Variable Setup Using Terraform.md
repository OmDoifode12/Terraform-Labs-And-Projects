# Day 35 – VPC Variable Setup Using Terraform

---

## Task Overview

As part of AWS infrastructure automation, the Nautilus DevOps team needs to create a Virtual Private Cloud (VPC) using Terraform while implementing variables for reusable and modular configurations.

For this task:

* Create a VPC with CIDR block:

```text id="vvs01"
10.0.0.0/16
```

* Store the VPC name:

```text id="vvs02"
devops-vpc
```

inside a Terraform variable named:

```text id="vvs03"
KKE_vpc
```

---

# Objective

| Property          | Value         |
| ----------------- | ------------- |
| Resource Type     | VPC           |
| VPC Name          | `devops-vpc`  |
| Variable Name     | `KKE_vpc`     |
| CIDR Block        | `10.0.0.0/16` |
| AWS Region        | `us-east-1`   |
| Provisioning Tool | Terraform     |

---

# Understanding Terraform Variables

Terraform variables help make infrastructure:

* Reusable
* Dynamic
* Modular
* Easier to manage

Instead of hardcoding values:

```hcl id="vvs04"
name = "devops-vpc"
```

we use:

```hcl id="vvs05"
name = var.KKE_vpc
```

---

# Why Use Variables?

Benefits of Terraform variables:

| Benefit     | Description                   |
| ----------- | ----------------------------- |
| Reusability | Reuse code easily             |
| Flexibility | Change values dynamically     |
| Modularity  | Cleaner infrastructure design |
| Scalability | Easier environment management |

# Step-by-Step Implementation

## Step 1: Navigate to Terraform Directory

```bash id="vvs07"
cd /home/bob/terraform
```

### Explanation

This is the Terraform working directory containing Terraform configuration files.

---

## Step 2: Create Terraform Files

```bash id="vvs08"
touch main.tf variables.tf
```

### Explanation

Creates the required Terraform files.

---

# Configuring variables.tf

## Step 3: Open variables.tf

```bash id="vvs09"
vi variables.tf
```

Press:

```text id="vvs10"
i
```

to enter INSERT mode.

---

## Step 4: Add Variable Configuration

```hcl id="vvs11"
variable "KKE_vpc" {
  default = "devops-vpc"
}
```

---

# Variable Explanation

| Block    | Purpose                     |
| -------- | --------------------------- |
| variable | Declares Terraform variable |
| KKE_vpc  | Variable name               |
| default  | Default variable value      |

---

## Step 5: Save variables.tf

Press:

```text id="vvs12"
ESC
```

Then type:

```text id="vvs13"
:wq
```

and press ENTER.

---

# Configuring main.tf

## Step 6: Open main.tf

```bash id="vvs14"
vi main.tf
```

Press:

```text id="vvs15"
i
```

to enter INSERT mode.

---

## Step 7: Add Terraform Configuration

```hcl id="vvs16"
resource "aws_vpc" "devops_vpc" {
  cidr_block = "10.0.0.0/16"

  tags = {
    Name = var.KKE_vpc
  }
}
```

---

# Code Explanation

| Block       | Purpose                       |
| ----------- | ----------------------------- |
| aws_vpc     | Creates AWS VPC               |
| cidr_block  | Defines VPC IP range          |
| tags        | Adds resource tags            |
| var.KKE_vpc | References Terraform variable |

---

# Understanding Variable Reference

```hcl id="vvs17"
var.KKE_vpc
```

Terraform reads the value from:

```text id="vvs18"
variables.tf
```

and assigns:

```text id="vvs19"
devops-vpc
```

as the VPC name.

---

# Understanding CIDR Block

```hcl id="vvs20"
10.0.0.0/16
```

Defines the available IP range inside the VPC.

Example:

| CIDR Block | Available IPs |
| ---------- | ------------- |
| /16        | 65,536 IPs    |

---

## Step 8: Save main.tf

Press:

```text id="vvs21"
ESC
```

Then type:

```text id="vvs22"
:wq
```

and press ENTER.

---

## Step 9: Initialize Terraform

```bash id="vvs23"
terraform init
```

### Explanation

Initializes Terraform and downloads required providers.

---

## Step 10: Validate Configuration

```bash id="vvs24"
terraform validate
```

### Explanation

Checks Terraform syntax and configuration correctness.

Expected Output:

```text id="vvs25"
Success! The configuration is valid.
```

---

## Step 11: Preview Infrastructure Changes

```bash id="vvs26"
terraform plan
```

### Explanation

Displays the VPC Terraform will create.

Expected Output:

```text id="vvs27"
1 to add
```

---

## Step 12: Apply Configuration

```bash id="vvs28"
terraform apply -auto-approve
```

### Explanation

Creates the AWS VPC.

Expected Output:

```text id="vvs29"
Apply complete! Resources: 1 added.
```

---

## Step 13: Verify VPC Creation

```bash id="vvs30"
aws ec2 describe-vpcs \
--filters "Name=tag:Name,Values=devops-vpc" \
--query "Vpcs[*].[VpcId,CidrBlock]" \
--output table
```

### Explanation

Verifies the VPC was created successfully.

Expected Output should contain:

```text id="vvs31"
devops-vpc
```

---

## Step 14: Verify Terraform State

Run:

```bash id="vvs32"
terraform plan
```

Expected Output:

```text id="vvs33"
No changes. Your infrastructure matches the configuration.
```

This confirms Terraform state and infrastructure are synchronized correctly.
