# Day 33 – Delete VPC Using Terraform

---

## Task Overview

As part of AWS infrastructure cleanup and optimization, the Nautilus DevOps team needs to remove an unused Virtual Private Cloud (VPC) using Terraform. However, the Terraform provisioning code must remain intact so the VPC can be recreated later if required.

For this task:

* Delete the VPC named:

```text id="dvp01"
datacenter-vpc
```

using Terraform.

* Keep the Terraform provisioning code unchanged.

---

# Objective

| Property          | Value            |
| ----------------- | ---------------- |
| Resource Type     | VPC              |
| VPC Name          | `datacenter-vpc` |
| CIDR Block        | `10.0.0.0/16`    |
| Action            | Delete VPC       |
| Keep Code         | Yes              |
| AWS Region        | `us-east-1`      |
| Provisioning Tool | Terraform        |

---

# Understanding Amazon VPC

Amazon VPC (Virtual Private Cloud) allows you to create isolated networks within AWS.

A VPC contains:

* Subnets
* Route tables
* Internet gateways
* Security groups
* EC2 instances

---

# Why Delete Unused VPCs?

Unused VPCs may create:

* Resource clutter
* Security concerns
* Unnecessary dependencies
* Management complexity

Cleanup improves:

* Infrastructure organization
* Operational efficiency
* Cloud governance

---

# Existing Terraform Configuration

```hcl id="dvp02"
resource "aws_vpc" "this" {
  cidr_block = "10.0.0.0/16"

  tags = {
    Name = "datacenter-vpc"
  }
}
```

# Important Requirement

The task specifically requires:

```text id="dvp03"
Delete the VPC but KEEP the Terraform code
```

This means:

| Action                | Required |
| --------------------- | -------- |
| Delete AWS Resource   | Yes      |
| Delete Terraform Code | No       |
| Keep main.tf          | Yes      |

# Architecture Overview

```text id="dvp04"
Terraform State → Targeted Destroy → VPC Deleted
```

# Step-by-Step Implementation

## Step 1: Navigate to Terraform Directory

```bash id="dvp05"
cd /home/bob/terraform
```

### Explanation

This is the Terraform working directory containing Terraform configuration files.

---

## Step 2: Verify Existing VPC

```bash id="dvp06"
aws ec2 describe-vpcs \
--filters "Name=tag:Name,Values=datacenter-vpc" \
--query "Vpcs[*].[VpcId,CidrBlock,State]" \
--output table
```

### Explanation

Checks whether the VPC exists before deletion.

Expected Output should contain:

```text id="dvp07"
datacenter-vpc
```

---

## Step 3: Initialize Terraform

```bash id="dvp08"
terraform init
```

### Explanation

Initializes Terraform and required providers.

---

## Step 4: Verify Terraform State

```bash id="dvp09"
terraform state list
```

### Explanation

Confirms the VPC resource exists in Terraform state.

Expected Output:

```text id="dvp10"
aws_vpc.this
```

---

## Step 5: Preview Targeted Destroy

```bash id="dvp11"
terraform plan -destroy -target=aws_vpc.this
```

### Explanation

Displays the Terraform plan for deleting only the VPC.

Expected Output:

```text id="dvp12"
Plan: 0 to add, 0 to change, 1 to destroy
```

---

## Step 6: Delete VPC

```bash id="dvp13"
terraform destroy -target=aws_vpc.this -auto-approve
```

### Explanation

Deletes only the VPC while keeping:

* Terraform configuration
* main.tf
* Provisioning code

Expected Output:

```text id="dvp14"
Destroy complete! Resources: 1 destroyed.
```

---

# IMPORTANT

Do NOT delete:

```text id="dvp15"
main.tf
```

The provisioning code must remain available for future deployments.

---

## Step 7: Verify VPC Deletion

```bash id="dvp16"
aws ec2 describe-vpcs \
--filters "Name=tag:Name,Values=datacenter-vpc" \
--query "Vpcs[*].[VpcId,State]" \
--output table
```

### Explanation

Verifies the VPC no longer exists.

Expected Output:

```text id="dvp17"
No VPCs returned
```

---

## Step 8: Verify Terraform State

```bash id="dvp18"
terraform state list
```

### Explanation

The VPC resource should no longer exist in Terraform state.

---

# Understanding Targeted Destroy

Terraform supports deleting individual resources using:

```bash id="dvp19"
terraform destroy -target=RESOURCE_NAME
```

This enables:

* Partial cleanup
* Controlled deletion
* Safe infrastructure management

---

# Understanding Terraform State

Terraform tracks infrastructure resources inside:

```text id="dvp20"
terraform.tfstate
```

After deletion:

```text id="dvp21"
Resource removed from Terraform state
```

but:

```text id="dvp22"
Terraform code remains unchanged
```

---

# Understanding VPC CIDR Block

```hcl id="dvp23"
cidr_block = "10.0.0.0/16"
```

Defines the IP range available inside the VPC.

Example:

| CIDR Block  | Available IP Range |
| ----------- | ------------------ |
| 10.0.0.0/16 | 65,536 IPs         |
