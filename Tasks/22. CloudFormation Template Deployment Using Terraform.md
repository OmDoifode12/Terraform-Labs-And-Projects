# Day 22 – CloudFormation Template Deployment Using Terraform

---

## Task Overview

As part of AWS infrastructure automation, the Nautilus DevOps team needs to deploy a CloudFormation stack using Terraform. The stack should provision an Amazon S3 bucket with versioning enabled.

For this task, create a CloudFormation stack named `datacenter-stack` that provisions an S3 bucket named `datacenter-bucket-3216` with versioning enabled.

---

# Objective

| Property          | Value                    |
| ----------------- | ------------------------ |
| Resource Type     | CloudFormation Stack     |
| Stack Name        | `datacenter-stack`       |
| S3 Bucket Name    | `datacenter-bucket-3216` |
| S3 Versioning     | Enabled                  |
| Provisioning Tool | Terraform                |
| AWS Service       | CloudFormation           |

---

# Understanding AWS CloudFormation

AWS CloudFormation is an Infrastructure as Code (IaC) service that helps automate AWS resource provisioning using templates.

CloudFormation templates can be written in:

* YAML
* JSON

Benefits:

* Automated deployments
* Consistent infrastructure
* Easy rollback support
* Repeatable provisioning

---

# Understanding CloudFormation Stacks

A CloudFormation Stack is a collection of AWS resources managed together.

Example:

```text id="cft01"
CloudFormation Template → Stack → AWS Resources
```

---

# Understanding S3 Versioning

Amazon S3 Versioning helps preserve, retrieve, and recover objects from accidental deletion or overwrites.

When enabled:

```text id="cft02"
Every object update creates a new version
```

Benefits:

* Data protection
* Backup recovery
* Accidental deletion recovery
* Object history tracking

---

# Architecture Overview

```text id="cft03"
Terraform → CloudFormation Stack → S3 Bucket with Versioning
```

```bash id="cft06"
vi main.tf
```

Press:

```text id="cft07"
i
```

to enter INSERT mode.

---

## Step 4: Add Terraform Configuration

```hcl id="cft08"
resource "aws_cloudformation_stack" "datacenter_stack" {
  name = "datacenter-stack"

  template_body = <<STACK
{
  "AWSTemplateFormatVersion": "2010-09-09",
  "Resources": {
    "S3Bucket": {
      "Type": "AWS::S3::Bucket",
      "Properties": {
        "BucketName": "datacenter-bucket-3216",
        "VersioningConfiguration": {
          "Status": "Enabled"
        }
      }
    }
  }
}
STACK
}
```

---

# Code Explanation

| Block                    | Purpose                          |
| ------------------------ | -------------------------------- |
| aws_cloudformation_stack | Creates CloudFormation stack     |
| name                     | Stack name                       |
| template_body            | Embedded CloudFormation template |
| AWS::S3::Bucket          | Creates S3 bucket                |
| VersioningConfiguration  | Enables bucket versioning        |

---

# Understanding template_body

```hcl id="cft09"
template_body
```

contains the actual CloudFormation JSON template.

Terraform submits this template to AWS CloudFormation for deployment.

---

# Understanding Versioning

```json id="cft10"

  "Status": "Enabled"
```

This enables S3 object versioning.
