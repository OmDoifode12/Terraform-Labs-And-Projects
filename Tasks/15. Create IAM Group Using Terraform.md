As part of their AWS cloud migration strategy, the Nautilus DevOps team is implementing Identity and Access Management (IAM) resources using Terraform. Managing permissions at scale becomes easier when users are organized into groups. IAM Groups allow administrators to assign permissions to multiple users at once, simplifying access management.

# Understanding IAM Groups

An IAM Group is a collection of IAM Users.

Instead of assigning permissions individually to every user, permissions can be assigned to a group and inherited by all users within that group.

Common Examples:

* Developers
* DevOps Team
* QA Team
* Security Team
* Administrators

Benefits:

* Simplified permission management
* Better access control
* Easier onboarding and offboarding
* Improved security governance

# Step-by-Step Implementation

## Step 1: Navigate to Terraform Directory

```bash
cd /home/bob/terraform
```

### Explanation

This is the Terraform working directory where infrastructure configuration files are stored.

---

## Step 2: Create main.tf

```bash
touch main.tf
```

### Explanation

Terraform resources are defined inside `.tf` files.

---

## Step 3: Add Terraform Configuration

```hcl
resource "aws_iam_group" "iamgroup_anita" {
  name = "iamgroup_anita"
}
```

### Explanation

| Parameter     | Description                  |
| ------------- | ---------------------------- |
| aws_iam_group | Creates an IAM Group         |
| name          | Specifies the IAM Group name |

---

## Step 4: Initialize Terraform

```bash
terraform init
```

### Explanation

Downloads required providers and initializes Terraform.

---

## Step 5: Validate Configuration

```bash
terraform validate
```

### Explanation

Checks Terraform syntax and configuration correctness.

---

## Step 6: Preview Changes

```bash
terraform plan
```

### Explanation

Displays resources Terraform will create before deployment.

---

## Step 7: Apply Configuration

```bash
terraform apply -auto-approve
```

### Explanation

Creates the IAM Group in AWS.

---

## Step 8: Verify IAM Group

```bash
aws iam list-groups
```

### Explanation

Lists all IAM Groups and verifies successful creation.

---

## Step 9: Verify Specific Group

```bash
aws iam get-group --group-name iamgroup_anita
```

### Explanation

Displays detailed information about the IAM Group.

# Best Practices

* Use IAM Groups to manage permissions efficiently.
* Follow the Principle of Least Privilege.
* Assign policies to groups instead of individual users.
* Use meaningful group names.
* Review group memberships regularly.

# Interview Questions & Answers

### 1. What is an IAM Group?

An IAM Group is a collection of IAM Users that share the same permissions.

---

### 2. Why use IAM Groups?

To simplify permission management by assigning permissions to multiple users at once.

---

### 3. Which Terraform resource creates an IAM Group?

```hcl
aws_iam_group
```

---

### 4. Can IAM Groups have credentials?

No. IAM Groups cannot authenticate or have credentials.

---

### 5. Can an IAM User belong to multiple Groups?

Yes. A single IAM User can be a member of multiple IAM Groups.

---

### 6. What is the benefit of assigning permissions to Groups?

It reduces administrative overhead and improves consistency.

---

### 7. What is the Principle of Least Privilege?

Granting only the minimum permissions required to perform a task.

# Real-World Example

Consider a company with 50 developers:

Instead of assigning permissions to each developer individually:

```text
Developer 1 → Permissions
Developer 2 → Permissions
Developer 3 → Permissions
...
Developer 50 → Permissions
```

Create a single group:

```text
Developers Group
```

Attach the required policy once, then add all developers to the group.

This approach is more secure, scalable, and easier to manage.
