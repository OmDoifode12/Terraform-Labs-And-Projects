# Day 06 – Create Elastic IP Using Terraform

---

## Task Overview

As part of the AWS cloud migration process, the Nautilus DevOps team is provisioning networking resources using Terraform. One of the key networking components required for cloud infrastructure is an Elastic IP (EIP), which provides a static public IPv4 address that can be associated with AWS resources such as EC2 instances.

For this task, allocate an Elastic IP address named `devops-eip` using Terraform.

---

# Objective

| Property          | Value            |
| ----------------- | ---------------- |
| Resource Type     | Elastic IP (EIP) |
| Name              | `devops-eip`     |
| Region            | `us-east-1`      |
| Provisioning Tool | Terraform        |

---

# Understanding Elastic IP

An Elastic IP (EIP) is a static public IPv4 address provided by AWS.

Benefits:

* Static public IP address
* Remains unchanged during instance restarts
* Can be reassociated with another EC2 instance
* Useful for production workloads

# Step-by-Step Implementation

## Step 1: Navigate to Terraform Directory

```bash
cd /home/bob/terraform
```

### Explanation

This is the working directory where Terraform configuration files are stored.

---

## Step 2: Create main.tf

```bash
touch main.tf
```

### Explanation

Terraform resources are defined inside `.tf` files.

---

## Step 3: Add Terraform Configuration

```hcl
provider "aws" {
  region = "us-east-1"
}

resource "aws_eip" "devops_eip" {
  domain = "vpc"

  tags = {
    Name = "devops-eip"
  }
}
```

### Explanation

| Parameter      | Description                 |
| -------------- | --------------------------- |
| aws_eip        | Creates an Elastic IP       |
| domain = "vpc" | Allocates EIP for VPC usage |
| tags           | Assigns a name to the EIP   |

---

## Step 4: Initialize Terraform

```bash
terraform init
```

### Explanation

Downloads required providers and initializes the Terraform project.

---

## Step 5: Validate Configuration

```bash
terraform validate
```

### Explanation

Checks Terraform syntax and configuration correctness.

---

## Step 6: Preview Infrastructure Changes

```bash
terraform plan
```

### Explanation

Displays resources Terraform will create before deployment.

---

## Step 7: Apply Configuration

```bash
terraform apply -auto-approve
```

### Explanation

Creates the Elastic IP address in AWS.

---

## Step 8: Verify Elastic IP

```bash
aws ec2 describe-addresses --region us-east-1
```

### Explanation

Lists allocated Elastic IP addresses and verifies successful creation.

An Elastic IP is a static public IPv4 address provided by AWS.

---

### 2. Why use an Elastic IP?

To maintain a consistent public IP address even if an EC2 instance is restarted or replaced.

### 3. Which Terraform resource creates an Elastic IP?

It allocates the Elastic IP for use within a VPC.

Yes, an Elastic IP can be reassociated with another instance.
