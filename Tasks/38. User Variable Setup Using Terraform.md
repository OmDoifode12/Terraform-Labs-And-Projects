# Day 38 – User Variable Setup Using Terraform

---

## Task Overview

As part of AWS Identity and Access Management (IAM) automation, the Nautilus DevOps team needs to create an IAM User using Terraform while implementing variables for reusable and modular configurations.

For this task:

* Create an AWS IAM User.
* Store the IAM User name:

```text id="uvs01"
iamuser_ravi
```

inside a Terraform variable named:

```text id="uvs02"
KKE_user
```

---

# Objective

| Property          | Value          |
| ----------------- | -------------- |
| Resource Type     | IAM User       |
| IAM User Name     | `iamuser_ravi` |
| Variable Name     | `KKE_user`     |
| AWS Service       | IAM            |
| AWS Region        | `us-east-1`    |
| Provisioning Tool | Terraform      |

---

# Understanding IAM Users

An IAM User is an identity created in AWS for:

* Human users
* Applications
* Automation systems

IAM Users can receive:

* AWS Console access
* Programmatic access
* Policies and permissions

---

# Why Use Variables in Terraform?

Terraform variables improve:

| Benefit     | Description                   |
| ----------- | ----------------------------- |
| Reusability | Reuse infrastructure easily   |
| Flexibility | Dynamic configuration values  |
| Modularity  | Cleaner Terraform code        |
| Scalability | Easier environment management |

Instead of hardcoding:

```hcl id="uvs03"
name = "iamuser_ravi"
```

we use:

```hcl id="uvs04"
name = var.KKE_user
```

---

# Architecture Overview

```text id="uvs05"
variables.tf → main.tf → IAM User
```

---

# Terraform File Structure

| File         | Purpose                    |
| ------------ | -------------------------- |
| variables.tf | Stores Terraform variables |
| main.tf      | Creates AWS resources      |

# Step-by-Step Implementation

## Step 1: Navigate to Terraform Directory

```bash id="uvs06"
cd /home/bob/terraform
```

### Explanation

This is the Terraform working directory containing Terraform configuration files.

---

## Step 2: Create Terraform Files

```bash id="uvs07"
touch main.tf variables.tf
```

### Explanation

Creates the required Terraform files.

---

# Configuring variables.tf

## Step 3: Open variables.tf

```bash id="uvs08"
vi variables.tf
```

Press:

```text id="uvs09"
i
```

to enter INSERT mode.

---

## Step 4: Add Variable Configuration

```hcl id="uvs10"
variable "KKE_user" {
  default = "iamuser_ravi"
}
```

---

# Variable Explanation

| Block    | Purpose                     |
| -------- | --------------------------- |
| variable | Declares Terraform variable |
| KKE_user | Variable name               |
| default  | Default variable value      |

---

## Step 5: Save variables.tf

Press:

```text id="uvs11"
ESC
```

Then type:

```text id="uvs12"
:wq
```

and press ENTER.

---

# Configuring main.tf

## Step 6: Open main.tf

```bash id="uvs13"
vi main.tf
```

Press:

```text id="uvs14"
i
```

to enter INSERT mode.

---

## Step 7: Add Terraform Configuration

```hcl id="uvs15"
resource "aws_iam_user" "iam_user" {
  name = var.KKE_user

  tags = {
    Name = var.KKE_user
  }
}
```

---

# Code Explanation

| Block        | Purpose                       |
| ------------ | ----------------------------- |
| aws_iam_user | Creates IAM User              |
| name         | IAM User name                 |
| tags         | Adds AWS resource tags        |
| var.KKE_user | References Terraform variable |

---

# Understanding Variable Reference

```hcl id="uvs16"
var.KKE_user
```

Terraform reads the value from:

```text id="uvs17"
variables.tf
```

and assigns:

```text id="uvs18"
iamuser_ravi
```

as the IAM User name.

---

# Understanding IAM User Tags

```hcl id="uvs19"
tags = {
  Name = var.KKE_user
}
```

Tags help organize and identify AWS resources.

Benefits:

* Better resource management
* Easier filtering
* Improved automation
* Cost tracking

---

## Step 8: Save main.tf

Press:

```text id="uvs20"
ESC
```

Then type:

```text id="uvs21"
:wq
```

and press ENTER.

---

## Step 9: Initialize Terraform

```bash id="uvs22"
terraform init
```

### Explanation

Initializes Terraform and downloads required providers.

---

## Step 10: Validate Configuration

```bash id="uvs23"
terraform validate
```

### Explanation

Checks Terraform syntax and configuration correctness.

Expected Output:

```text id="uvs24"
Success! The configuration is valid.
```

---

## Step 11: Preview Infrastructure Changes

```bash id="uvs25"
terraform plan
```

### Explanation

Displays the IAM User Terraform will create.

Expected Output:

```text id="uvs26"
1 to add
```

---

## Step 12: Apply Configuration

```bash id="uvs27"
terraform apply -auto-approve
```

### Explanation

Creates the AWS IAM User.

Expected Output:

```text id="uvs28"
Apply complete! Resources: 1 added.
```

---

## Step 13: Verify IAM User Creation

```bash id="uvs29"
aws iam get-user \
--user-name iamuser_ravi
```

### Explanation

Verifies the IAM User was created successfully.

Expected Output should contain:

```text id="uvs30"
iamuser_ravi
```

---

## Step 14: Verify Terraform State

Run:

```bash id="uvs31"
terraform plan
```

Expected Output:

```text id="uvs32"
No changes. Your infrastructure matches the configuration.
```

This confirms Terraform state and infrastructure are synchronized correctly.
