# Understanding Private S3 Buckets

By default, S3 buckets are private. However, AWS provides additional Public Access Block settings to prevent accidental exposure of sensitive data.

Benefits of Private Buckets:

* Enhanced security
* Protection of sensitive information
* Compliance requirements
* Reduced risk of data leaks

---

# Understanding Public Access Block

AWS provides four settings to restrict public access:

| Setting               | Purpose                          |
| --------------------- | -------------------------------- |
| BlockPublicAcls       | Blocks public ACLs               |
| IgnorePublicAcls      | Ignores existing public ACLs     |
| BlockPublicPolicy     | Blocks public bucket policies    |
| RestrictPublicBuckets | Restricts public access entirely |

For maximum security, all four settings should be enabled.

# Step-by-Step Implementation

## Step 1: Navigate to Terraform Directory

```bash
cd /home/bob/terraform
```

### Explanation

This is the Terraform working directory where infrastructure configuration files are stored.

---

## Step 2: Create main.tf

```bash
touch main.tf
```

### Explanation

Terraform resources are defined inside `.tf` files.

---

## Step 3: Add Terraform Configuration

```hcl
resource "aws_s3_bucket" "devops_bucket" {
  bucket = "devops-s3-23217"
}

resource "aws_s3_bucket_public_access_block" "devops_private_access" {
  bucket = aws_s3_bucket.devops_bucket.id

  block_public_acls       = true
  block_public_policy     = true
  ignore_public_acls      = true
  restrict_public_buckets = true
}
```

### Explanation

| Resource                          | Purpose                  |
| --------------------------------- | ------------------------ |
| aws_s3_bucket                     | Creates the S3 bucket    |
| aws_s3_bucket_public_access_block | Blocks all public access |

---

## Step 7: Apply Configuration


---

## Step 8: Verify Bucket Creation

```bash

```

### Explanation

Lists all available S3 buckets and confirms successful creation.

---

## Step 9: Verify Public Access Block

```bash
aws --endpoint-url=http://aws:4566 s3api get-public-access-block \

```

### Explanation

```text

```

Expected Output:

# Best Practices

* Keep buckets private by default.
* Enable all Public Access Block settings.
* Use IAM policies for controlled access.
* Encrypt sensitive data at rest.
* Regularly audit bucket permissions.

# Key Learnings

* S3 buckets can be configured as private or public.
* Public Access Block helps prevent accidental exposure.
* Terraform automates secure storage provisioning.
* Security should be implemented by default.
