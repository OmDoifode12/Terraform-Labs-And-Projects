# Day 09 – Create EBS Volume Using Terraform

---

## Task Overview

As part of the AWS cloud migration initiative, the Nautilus DevOps team is provisioning storage resources using Terraform. Amazon Elastic Block Store (EBS) provides persistent block-level storage for EC2 instances and is commonly used for databases, application storage, and backups.

For this task, create an EBS volume named `devops-volume` with a size of **2 GiB** and volume type **gp3** in the **us-east-1** region using Terraform.

Common Use Cases:

* Operating System disks
* Databases
* Application storage
* Backup storage
* High-performance workloads

Benefits:

* Persistent storage

# Step-by-Step Implementation

## Step 1: Navigate to Terraform Directory

```bash
cd /home/bob/terraform
```

### Explanation

This is the Terraform working directory where infrastructure configuration files are stored.

---

## Step 2: Create main.tf

```bash
touch main.tf
```

### Explanation

Terraform resources are defined inside `.tf` files.

---

## Step 3: Add Terraform Configuration

```hcl
provider "aws" {
  region = "us-east-1"
}

resource "aws_ebs_volume" "devops_volume" {
  availability_zone = "us-east-1a"
  size              = 2
  type              = "gp3"

  tags = {
    Name = "devops-volume"
  }
}
```

### Explanation

| Parameter         | Description                     |
| ----------------- | ------------------------------- |
| aws_ebs_volume    | Creates an EBS volume           |
| availability_zone | Specifies the Availability Zone |
| size              | Volume size in GiB              |
| type              | EBS volume type                 |
| tags              | Assigns a name to the volume    |

---

## Step 4: Initialize Terraform

```bash
terraform init
```

### Explanation

Downloads required providers and initializes Terraform.

---

## Step 5: Validate Configuration

```bash
terraform validate
```

### Explanation

Checks Terraform syntax and configuration correctness.

---

## Step 6: Preview Changes

```bash
terraform plan
```

### Explanation

Displays resources Terraform will create before deployment.

---

## Step 7: Apply Configuration

```bash
terraform apply -auto-approve
```

### Explanation

Creates the EBS volume in AWS.

---

## Step 8: Verify EBS Volume

```bash
aws ec2 describe-volumes --region us-east-1
```

### Explanation

Lists EBS volumes and verifies successful creation.

---

## Step 9: Verify Volume Details

```bash
aws ec2 describe-volumes \
--query "Volumes[*].[VolumeId,Size,VolumeType]" \
--output table
```

### Explanation

Verify:

| Property | Expected Value |
| -------- | -------------- |
| Size     | 2              |
| Type     | gp3            |

# Interview Questions & Answers

### 1. What is Amazon EBS?

Amazon Elastic Block Store (EBS) is a block-level storage service for EC2 instances.

---

### 2. Is EBS storage persistent?

Yes. Data remains available even if the EC2 instance is stopped.

---

### 3. What is a gp3 volume?

gp3 is AWS's latest generation General Purpose SSD volume that provides high performance at lower cost.

---

### 4. Which Terraform resource creates an EBS volume?

```hcl
aws_ebs_volume
```

---

### 5. Can an EBS volume exist without an EC2 instance?

Yes. EBS volumes can be created independently and attached later.

---

### 6. What is the difference between gp2 and gp3?

| gp2                        | gp3                     |
| -------------------------- | ----------------------- |
| Performance linked to size | Independent performance |
| Older generation           | Newer generation        |
| Less cost-efficient        | More cost-efficient     |

---

### 7. What command initializes Terraform?

```bash
terraform init
```

---

### 8. What command previews infrastructure changes?

```bash
terraform plan
```

---

### 9. Can EBS volumes be backed up?

Yes, using EBS Snapshots.
