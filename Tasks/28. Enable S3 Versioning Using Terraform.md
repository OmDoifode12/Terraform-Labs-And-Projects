# Day 28 – Enable S3 Versioning Using Terraform

---

## Task Overview

As part of AWS data protection and recovery implementation, the DevOps team needs to enable versioning for an existing Amazon S3 bucket using Terraform. S3 Versioning helps protect data from accidental deletion, overwrites, and corruption by maintaining multiple versions of objects.

For this task, enable versioning for the S3 bucket:

```text id="s3v01"
datacenter-s3-7899
```

using Terraform.

---

# Objective

| Property          | Value                |
| ----------------- | -------------------- |
| Resource Type     | S3 Bucket            |
| Bucket Name       | `datacenter-s3-7899` |
| Versioning        | Enabled              |
| AWS Service       | Amazon S3            |
| Provisioning Tool | Terraform            |

---

# Understanding Amazon S3 Versioning

Amazon S3 Versioning allows multiple versions of an object to exist in the same bucket.

Benefits:

* Data recovery
* Accidental deletion protection
* Object history tracking
* Backup safety
* Rollback capability

---

# Why Enable Versioning?

Without versioning:

```text id="s3v02"
New upload replaces old object permanently
```

With versioning:

```text id="s3v03"
Every object update creates a new version
```

---

# Example Workflow

```text id="s3v04"
file.txt v1 → file.txt v2 → file.txt v3
```

Older versions remain recoverable.

---

## Step 2: Open main.tf

```bash id="s3v09"
vi main.tf
```

Press:

```text id="s3v10"
i
```

to enter INSERT mode.

---

## Step 3: Add Versioning Block

Update the existing S3 bucket resource:

```hcl id="s3v11"
resource "aws_s3_bucket" "s3_ran_bucket" {
  bucket = "datacenter-s3-7899"
  acl    = "private"

  versioning {
    enabled = true
  }

  tags = {
    Name = "datacenter-s3-7899"
  }
}
```

---

# Code Explanation

| Block         | Purpose                       |
| ------------- | ----------------------------- |
| aws_s3_bucket | Creates and manages S3 bucket |
| bucket        | Bucket name                   |
| acl           | Bucket access control         |
| versioning    | Enables S3 object versioning  |
| enabled       | Turns versioning ON           |
| tags          | Adds resource tags            |

---

# Understanding Versioning Block

```hcl id="s3v12"
versioning {
  enabled = true
}
```

This enables:

```text id="s3v13"
Object version tracking
```

for all objects stored in the bucket.



| Delete object   | Delete marker added          |
| Restore file    | Previous version recoverable |

* Enable encryption for sensitive data
