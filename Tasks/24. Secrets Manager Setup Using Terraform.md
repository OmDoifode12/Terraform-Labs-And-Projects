# Day 24 – Secrets Manager Setup Using Terraform

---

## Task Overview

As part of AWS infrastructure automation and security management, the Nautilus DevOps team needs to securely store sensitive credentials using AWS Secrets Manager. Secrets Manager helps securely manage, rotate, and retrieve application secrets such as usernames, passwords, API keys, and database credentials.

For this task, create a secret named `devops-secret` containing:

* Username: `admin`
* Password: `Namin123`

using Terraform.

# Understanding AWS Secrets Manager

AWS Secrets Manager is a managed AWS service used to securely store and retrieve secrets.

Secrets can include:

* Database credentials
* API keys
* Tokens
* Application passwords
* SSH keys

Benefits:

* Secure storage
* Encryption
* Automatic rotation
* Fine-grained access control
* Centralized secret management

```bash id="sm06"
vi main.tf
```

Press:

```text id="sm07"
i
```

to enter INSERT mode.

---

## Step 4: Add Terraform Configuration

```hcl id="sm08"
resource "aws_secretsmanager_secret" "devops_secret" {
  name = "devops-secret"
}

resource "aws_secretsmanager_secret_version" "secret_value" {
  secret_id = aws_secretsmanager_secret.devops_secret.id

  secret_string = jsonencode({
    username = "admin"
    password = "Namin123"
  })
}
```

---

# Code Explanation

| Block                             | Purpose                      |
| --------------------------------- | ---------------------------- |
| aws_secretsmanager_secret         | Creates secret container     |
| name                              | Secret name                  |
| aws_secretsmanager_secret_version | Stores secret value          |
| secret_id                         | Associates value with secret |
| secret_string                     | Stores JSON secret data      |

---

# Understanding Secret Versions

Secrets Manager uses versions to track updates.

Each time the secret value changes:

```text id="sm09"
A new secret version is created
```

Benefits:

* Secret history tracking
* Rollback support
* Version management

---

# Understanding jsonencode()

```hcl id="sm10"
jsonencode()
```

converts Terraform data into valid JSON format.

This is required for structured secret storage.

## Step 11: Retrieve Secret Value


# Best Practices

* Never hardcode secrets in applications
* Use IAM least-privilege policies
* Enable automatic rotation in production
* Encrypt sensitive data
* Regularly rotate passwords and API keys
