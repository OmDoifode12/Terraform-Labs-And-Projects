# Day 25 – Change Instance Type Using Terraform

---

## Task Overview

As part of AWS infrastructure optimization, the Nautilus DevOps team identified that an EC2 instance was underutilized and needed resizing to reduce resource consumption and cost. Terraform is used to manage the EC2 infrastructure and update the instance configuration.

For this task, change the EC2 instance type of `datacenter-ec2` from:

```text id="cit01"
t2.micro → t2.nano
```

using Terraform.

Ensure the instance remains in the:

```text id="cit02"
running
```

state after modification.

---

# Objective

| Property          | Value            |
| ----------------- | ---------------- |
| Resource Type     | EC2 Instance     |
| Instance Name     | `datacenter-ec2` |
| Old Instance Type | `t2.micro`       |
| New Instance Type | `t2.nano`        |
| AWS Region        | `us-east-1`      |
| Provisioning Tool | Terraform        |

---

# Understanding EC2 Instance Types

EC2 instance types define:

* CPU capacity
* Memory
* Network performance
* Storage throughput

Example:

| Instance Type | Usage                    |
| ------------- | ------------------------ |
| t2.micro      | Small workloads          |
| t2.nano       | Very lightweight systems |

---

# Why Resize EC2 Instances?

Resizing helps optimize:

* Infrastructure cost
* CPU utilization
* Resource efficiency
* Performance requirements

Example:

```text id="cit03"
Underutilized Instance → Smaller Instance Type
```

---

# Understanding Terraform State

Terraform keeps track of infrastructure using:

```text id="cit04"
terraform.tfstate
```

When instance properties change:

```text id="cit05"
Terraform updates the infrastructure automatically
```

# Architecture Overview

```text id="cit06"
Terraform → EC2 Instance → Instance Type Updated
```

# Step-by-Step Implementation

## Step 1: Navigate to Terraform Directory

```bash id="cit08"
cd /home/bob/terraform
```

### Explanation

This is the Terraform working directory containing Terraform configuration files.

---

## Step 2: Verify Existing EC2 Instance

```bash id="cit09"
aws ec2 describe-instances \
--filters "Name=tag:Name,Values=datacenter-ec2" \
--query "Reservations[*].Instances[*].[InstanceId,InstanceType,State.Name]" \
--output table
```

### Explanation

Displays:

* Instance ID
* Current instance type
* Instance state

Expected Output:

```text id="cit10"
t2.micro
```

---

## Step 3: Open main.tf

```bash id="cit11"
vi main.tf
```

Press:

```text id="cit12"
i
```

to enter INSERT mode.

---

## Step 4: Update Instance Type

Locate:

```hcl id="cit13"
instance_type = "t2.micro"
```

Replace with:

```hcl id="cit14"
instance_type = "t2.nano"
```

---

# Example Terraform Configuration

```hcl id="cit15"
resource "aws_instance" "datacenter_ec2" {
  ami           = "ami-0c101f26f147fa7fd"
  instance_type = "t2.nano"

  tags = {
    Name = "datacenter-ec2"
  }
}
```

---

# Code Explanation

| Block         | Purpose                   |
| ------------- | ------------------------- |
| aws_instance  | Manages EC2 instance      |
| ami           | Amazon Machine Image      |
| instance_type | Defines EC2 instance size |
| tags          | Adds resource tags        |

---

# Understanding Instance Replacement

Some EC2 property changes can trigger:

```text id="cit16"
Stop → Modify → Start
```

Terraform handles this automatically.

## Step 5: Save File

```text id="cit17"
ESC
```

and press ENTER.

## Step 6: Initialize Terraform

```bash id="cit19"
terraform init
```

### Explanation

Initializes Terraform and required providers.

---

## Step 7: Validate Configuration

```bash id="cit20"
terraform validate
```

### Explanation

Checks Terraform syntax and configuration validity.

Expected Output:

```text id="cit21"
Success! The configuration is valid.
```

---

## Step 8: Preview Infrastructure Changes

```bash id="cit22"
terraform plan
```

### Explanation

Shows the instance type modification Terraform will perform.

Expected Change:

```text id="cit23"
t2.micro -> t2.nano
```

---

## Step 9: Apply Changes

```bash id="cit24"
terraform apply -auto-approve
```

### Explanation

Terraform updates the EC2 instance type.

Expected Output:

```text id="cit25"
Apply complete!
```

---

## Step 10: Verify Updated Instance Type

```bash id="cit26"
aws ec2 describe-instances \
--filters "Name=tag:Name,Values=datacenter-ec2" \
--query "Reservations[*].Instances[*].[InstanceId,InstanceType,State.Name]" \
--output table
```

### Explanation

Verify:

| Property      | Expected Value |
| ------------- | -------------- |
| Instance Type | `t2.nano`      |
| State         | `running`      |

---

## Step 11: Verify Terraform State

Run:

```bash id="cit27"
terraform plan
```

Expected Output:

```text id="cit28"
No changes. Your infrastructure matches the configuration.
```

This confirms Terraform state and AWS infrastructure are synchronized correctly.

---

# Understanding Terraform Plan Output

Terraform compares:

```text id="cit29"
Terraform Configuration ↔ Actual Infrastructure
```

If both match:

```text id="cit30"
No changes required
```
