# Day 37 – Elastic IP Variable Setup Using Terraform

---

## Task Overview

As part of AWS cloud migration and infrastructure automation, the Nautilus DevOps team needs to allocate an Elastic IP using Terraform while implementing variables for reusable and modular configurations.

For this task:

* Create an AWS Elastic IP.
* Store the Elastic IP name:

```text id="eipv01"
devops-eip
```

inside a Terraform variable named:

```text id="eipv02"
KKE_eip
```

---

# Objective

| Property          | Value        |
| ----------------- | ------------ |
| Resource Type     | Elastic IP   |
| Elastic IP Name   | `devops-eip` |
| Variable Name     | `KKE_eip`    |
| AWS Region        | `us-east-1`  |
| Provisioning Tool | Terraform    |

---

# Understanding Elastic IP

An Elastic IP (EIP) is a static public IPv4 address provided by AWS.

Benefits:

* Persistent public IP
* Stable external connectivity
* Supports failover
* Useful for production workloads

---

# Why Use Variables in Terraform?

Terraform variables improve:

| Benefit     | Description                   |
| ----------- | ----------------------------- |
| Reusability | Reuse infrastructure easily   |
| Flexibility | Dynamic configuration values  |
| Modularity  | Cleaner Terraform code        |
| Scalability | Easier environment management |

Instead of hardcoding:

```hcl id="eipv03"
Name = "devops-eip"
```

we use:

```hcl id="eipv04"
Name = var.KKE_eip
```

---

# Architecture Overview

```text id="eipv05"
variables.tf → main.tf → Elastic IP
```

# Terraform File Structure

| File         | Purpose                    |
| ------------ | -------------------------- |
| variables.tf | Stores Terraform variables |
| main.tf      | Creates AWS resources      |

# Step-by-Step Implementation

## Step 1: Navigate to Terraform Directory

```bash id="eipv06"
cd /home/bob/terraform
```

### Explanation

This is the Terraform working directory containing Terraform configuration files.

---

## Step 2: Create Terraform Files

```bash id="eipv07"
touch main.tf variables.tf
```

### Explanation

Creates the required Terraform files.

---

# Configuring variables.tf

## Step 3: Open variables.tf

```bash id="eipv08"
vi variables.tf
```

Press:

```text id="eipv09"
i
```

to enter INSERT mode.

---

## Step 4: Add Variable Configuration

```hcl id="eipv10"
variable "KKE_eip" {
  default = "devops-eip"
}
```

---

# Variable Explanation

| Block    | Purpose                     |
| -------- | --------------------------- |
| variable | Declares Terraform variable |
| KKE_eip  | Variable name               |
| default  | Default variable value      |

---

## Step 5: Save variables.tf

Press:

```text id="eipv11"
ESC
```

Then type:

```text id="eipv12"
:wq
```

and press ENTER.

---

# Configuring main.tf

## Step 6: Open main.tf

```bash id="eipv13"
vi main.tf
```

Press:

```text id="eipv14"
i
```

to enter INSERT mode.

---

## Step 7: Add Terraform Configuration

```hcl id="eipv15"
resource "aws_eip" "devops_eip" {

  tags = {
    Name = var.KKE_eip
  }
}
```

---

# Code Explanation

| Block       | Purpose                       |
| ----------- | ----------------------------- |
| aws_eip     | Creates Elastic IP            |
| tags        | Adds AWS resource tags        |
| var.KKE_eip | References Terraform variable |

---

# Understanding Variable Reference

```hcl id="eipv16"
var.KKE_eip
```

Terraform reads the value from:

```text id="eipv17"
variables.tf
```

and assigns:

```text id="eipv18"
devops-eip
```

as the Elastic IP name tag.

---

# Understanding Elastic IP Allocation

When Terraform creates:

```hcl id="eipv19"
aws_eip
```

AWS allocates:

```text id="eipv20"
A static public IPv4 address
```

that remains persistent until released.

---

## Step 8: Save main.tf

Press:

```text id="eipv21"
ESC
```

Then type:

```text id="eipv22"
:wq
```

and press ENTER.

---

## Step 9: Initialize Terraform

```bash id="eipv23"
terraform init
```

### Explanation

Initializes Terraform and downloads required providers.

---

## Step 10: Validate Configuration

```bash id="eipv24"
terraform validate
```

### Explanation

Checks Terraform syntax and configuration correctness.

Expected Output:

```text id="eipv25"
Success! The configuration is valid.
```

---

## Step 11: Preview Infrastructure Changes

```bash id="eipv26"
terraform plan
```

### Explanation

Displays the Elastic IP Terraform will create.

Expected Output:

```text id="eipv27"
1 to add
```

---

## Step 12: Apply Configuration

```bash id="eipv28"
terraform apply -auto-approve
```

### Explanation

Creates the AWS Elastic IP.

Expected Output:

```text id="eipv29"
Apply complete! Resources: 1 added.
```

---

## Step 13: Verify Elastic IP Creation

```bash id="eipv30"
aws ec2 describe-addresses \
--query "Addresses[*].[AllocationId,PublicIp,Tags]" \
--output table
```

### Explanation

Verifies the Elastic IP was created successfully.

Expected Output should contain:

```text id="eipv31"
devops-eip
```

---

# Note About Empty Output

If you see:

```text id="eipv32"
list index out of range
```

it usually means:

* No Elastic IP exists yet
* The query format returned empty results

Run:

```bash id="eipv33"
aws ec2 describe-addresses
```

instead to verify manually.

---

## Step 14: Verify Terraform State

Run:

```bash id="eipv34"
terraform plan
```

Expected Output:

```text id="eipv35"
No changes. Your infrastructure matches the configuration.
```

This confirms Terraform state and infrastructure are synchronized correctly.
