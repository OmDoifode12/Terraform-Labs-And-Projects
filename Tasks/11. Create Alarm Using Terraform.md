# Day 11 – Create CloudWatch Alarm Using Terraform

---

## Task Overview

Monitoring is a critical component of any cloud infrastructure. As part of their AWS migration strategy, the Nautilus DevOps team is implementing proactive monitoring to ensure application availability and performance.

For this task, create a CloudWatch alarm named `xfusion-alarm` that monitors the CPU utilization of an EC2 instance. The alarm should trigger whenever CPU usage exceeds **80%** for a period of **5 minutes**.

---

# Objective

| Property           | Value            |
| ------------------ | ---------------- |
| Resource Type      | CloudWatch Alarm |
| Alarm Name         | `xfusion-alarm`  |
| Metric             | CPUUtilization   |
| Threshold          | 80%              |
| Evaluation Period  | 5 Minutes        |
| Evaluation Periods | 1                |
| Service            | EC2              |
| Provisioning Tool  | Terraform        |

# Understanding CloudWatch Alarms

Amazon CloudWatch Alarms monitor AWS resources and trigger actions when predefined thresholds are crossed.

Common Monitoring Metrics:

* CPU Utilization
* Memory Usage
* Disk Usage
* Network Traffic
* Application Logs

Benefits:

* Proactive monitoring
* Automated notifications
* Auto Scaling integration
* Performance tracking
* Incident response

# Step-by-Step Implementation

## Step 1: Navigate to Terraform Directory

```bash
cd /home/bob/terraform
```

### Explanation

This is the Terraform working directory where infrastructure configuration files are stored.

---

## Step 2: Identify EC2 Instance

Run:

```bash
aws ec2 describe-instances \
--region us-east-1
```

### Explanation

This command lists EC2 instances and helps identify the Instance ID required for monitoring.

---

## Step 3: Create main.tf

```bash
touch main.tf
```

### Explanation

Terraform resources are defined inside `.tf` files.

---

## Step 4: Add Terraform Configuration

```hcl
resource "aws_cloudwatch_metric_alarm" "xfusion_alarm" {
  alarm_name          = "xfusion-alarm"
  comparison_operator = "GreaterThanThreshold"
  evaluation_periods  = 1

  metric_name = "CPUUtilization"
  namespace   = "AWS/EC2"
  period      = 300
  statistic   = "Average"

  threshold = 80

  dimensions = {
    InstanceId = "i-xxxxxxxxxxxxxxxxx"
  }

  alarm_description = "Alarm when CPU utilization exceeds 80%"
}
```

### Explanation

| Parameter           | Description                  |
| ------------------- | ---------------------------- |
| alarm_name          | Name of the CloudWatch alarm |
| comparison_operator | Comparison condition         |
| evaluation_periods  | Number of periods evaluated  |
| metric_name         | Metric being monitored       |
| namespace           | AWS service namespace        |
| period              | Evaluation period in seconds |
| statistic           | Aggregation method           |
| threshold           | CPU threshold value          |
| dimensions          | Target EC2 instance          |
| alarm_description   | Alarm description            |

---

## Step 5: Replace Instance ID

Replace:

```text
i-xxxxxxxxxxxxxxxxx
```

with the actual EC2 Instance ID.

Example:

```hcl
dimensions = {
  InstanceId = "i-0abc123456789def0"
}
```

### Explanation

CloudWatch needs the specific EC2 instance ID to monitor CPU utilization.

---

## Step 6: Initialize Terraform

```bash
terraform init
```

### Explanation

Downloads required providers and initializes Terraform.

---

## Step 7: Validate Configuration

```bash
terraform validate
```

### Explanation

Checks Terraform syntax and configuration correctness.

---

## Step 8: Preview Changes

```bash
terraform plan
```

### Explanation

Displays resources Terraform will create before deployment.

---

## Step 9: Apply Configuration

```bash
terraform apply -auto-approve
```

### Explanation

Creates the CloudWatch alarm in AWS.

---

## Step 10: Verify Alarm

```bash
aws cloudwatch describe-alarms \
--alarm-names "xfusion-alarm"
```

### Explanation

Lists alarm details and verifies successful creation.

---

## Step 11: Check Alarm State

```bash
aws cloudwatch describe-alarms \
--alarm-names "xfusion-alarm" \
--query "MetricAlarms[*].[AlarmName,StateValue]"
```

### Explanation

Expected states:

| State             | Meaning                   |
| ----------------- | ------------------------- |
| OK                | Metric below threshold    |
| ALARM             | Threshold exceeded        |
| INSUFFICIENT_DATA | Not enough data available |
