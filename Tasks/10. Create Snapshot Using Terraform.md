# Day 10 – Create Snapshot Using Terraform

---

## Task Overview

As part of the AWS cloud migration and backup strategy, the Nautilus DevOps team is implementing automated data protection mechanisms. Amazon EBS Snapshots provide point-in-time backups of EBS volumes and are commonly used for disaster recovery, backup automation, and data restoration.

For this task, create a snapshot named `nautilus-vol-ss` from an existing EBS volume named `nautilus-vol` using Terraform.

---

# Objective

| Property          | Value               |
| ----------------- | ------------------- |
| Resource Type     | EBS Snapshot        |
| Source Volume     | `nautilus-vol`      |
| Snapshot Name     | `nautilus-vol-ss`   |
| Description       | `Nautilus Snapshot` |
| Region            | `us-east-1`         |
| Provisioning Tool | Terraform           |

---

# Understanding EBS Snapshots

An EBS Snapshot is a point-in-time backup of an EBS volume stored in Amazon S3.

Benefits:

* Data backup and recovery
* Disaster recovery
* Volume cloning
* Long-term storage
* Infrastructure migration

# Step-by-Step Implementation

## Step 1: Navigate to Terraform Directory

```bash
cd /home/bob/terraform
```

### Explanation

This is the Terraform working directory where infrastructure configuration files are stored.

---

## Step 2: Identify Existing Volume

Run:

```bash
aws ec2 describe-volumes --region us-east-1
```

### Explanation

Lists available EBS volumes and helps identify the source volume.

---

## Step 3: Create main.tf

```bash
touch main.tf
```

### Explanation

Terraform resources are defined inside `.tf` files.

---

## Step 4: Add Terraform Configuration

```hcl
data "aws_ebs_volume" "nautilus_vol" {
  filter {
    name   = "tag:Name"
    values = ["nautilus-vol"]
  }
}

resource "aws_ebs_snapshot" "nautilus_snapshot" {
  volume_id   = data.aws_ebs_volume.nautilus_vol.id
  description = "Nautilus Snapshot"

  tags = {
    Name = "nautilus-vol-ss"
  }
}
```

### Explanation

| Parameter        | Description                 |
| ---------------- | --------------------------- |
| aws_ebs_volume   | Finds existing EBS volume   |
| tag:Name         | Searches volume by Name tag |
| aws_ebs_snapshot | Creates EBS snapshot        |
| volume_id        | Source volume ID            |
| description      | Snapshot description        |
| tags             | Assigns snapshot name       |

---

## Step 5: Initialize Terraform

```bash
terraform init
```

### Explanation

Downloads required providers and initializes Terraform.

---

## Step 6: Validate Configuration

```bash
terraform validate
```

### Explanation

Checks Terraform syntax and configuration correctness.

---

## Step 7: Preview Changes

```bash
terraform plan
```

### Explanation

Displays resources Terraform will create before deployment.

---

## Step 8: Apply Configuration

```bash
terraform apply -auto-approve
```

### Explanation

Creates the EBS snapshot from the specified volume.

---

## Step 9: Verify Snapshot Creation

```bash
aws ec2 describe-snapshots \
--owner-ids self \
--region us-east-1
```

### Explanation

Lists available snapshots and verifies successful creation.

---

## Step 10: Check Snapshot Status

```bash
aws ec2 describe-snapshots \
--owner-ids self \
--query "Snapshots[*].[SnapshotId,State]" \
--output table
```

### Explanation

Ensure snapshot state becomes:

```text
completed
```

before considering the backup process successful.

# Interview Questions & Answers

### 1. What is an EBS Snapshot?

An EBS Snapshot is a point-in-time backup of an EBS volume.

---

### 2. Where are EBS Snapshots stored?

Amazon S3.

---

### 3. Which Terraform resource creates a snapshot?

```hcl
aws_ebs_snapshot
```

---

### 4. Can a new volume be created from a snapshot?

Yes, EBS volumes can be restored from snapshots.

---

### 5. Are EBS Snapshots incremental?

Yes. After the first snapshot, only changed blocks are stored.

---

### 6. What is the purpose of `volume_id`?

It specifies the source EBS volume for the snapshot.

---

### 7. What snapshot state indicates success?

```text
completed
```
