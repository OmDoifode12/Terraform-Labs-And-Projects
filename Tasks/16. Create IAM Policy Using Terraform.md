# Day 16 – Create IAM Policy Using Terraform

---

## Task Overview

Identity and Access Management (IAM) is one of the most critical AWS services used to manage permissions and secure cloud resources. As part of the AWS infrastructure setup, the Nautilus DevOps team needs to create an IAM Policy that provides read-only access to Amazon EC2 resources.

For this task, create an IAM Policy named `iampolicy_siva` using Terraform. The policy should allow users to view EC2 instances, AMIs, snapshots, volumes, and related EC2 resources without allowing modification access.

# Understanding IAM Policies

An IAM Policy is a JSON document that defines permissions in AWS.

Policies determine:

* What actions are allowed
* Which resources can be accessed
* Whether access is allowed or denied

---

# Understanding Read-Only Access

Read-only access allows users to:

✅ View EC2 Instances
✅ View AMIs
✅ View Snapshots
✅ View Security Groups
✅ View Volumes

But users CANNOT:

❌ Launch instances
❌ Delete resources
❌ Modify configurations

## Step 4: Add Terraform Configuration

```hcl id="7blhn6"
resource "aws_iam_policy" "iampolicy_siva" {
  name        = "iampolicy_siva"
  description = "Read-only access to EC2 console"

  policy = jsonencode({
    Version = "2012-10-17"

    Statement = [
      {
        Effect = "Allow"

        Action = [
          "ec2:DescribeInstances",
          "ec2:DescribeImages",
          "ec2:DescribeSnapshots",
          "ec2:DescribeVolumes",
          "ec2:DescribeSecurityGroups",
          "ec2:DescribeKeyPairs",
          "ec2:DescribeTags"
        ]

        Resource = "*"
      }
    ]
  })
}
```

---

# Code Explanation

| Block          | Purpose                           |
| -------------- | --------------------------------- |
| aws_iam_policy | Creates IAM Policy                |
| name           | Policy name                       |
| description    | Policy description                |
| jsonencode     | Converts Terraform code into JSON |
| Effect = Allow | Grants permissions                |
| Action         | Defines allowed EC2 actions       |
| Resource = "*" | Applies to all EC2 resources      |

---

# Understanding EC2 Describe Actions

| Action                     | Purpose              |
| -------------------------- | -------------------- |
| ec2:DescribeInstances      | View EC2 instances   |
| ec2:DescribeImages         | View AMIs            |

| ec2:DescribeVolumes        | View EBS volumes     |
| ec2:DescribeSecurityGroups | View security groups |
| ec2:DescribeKeyPairs       | View key pairs       |
| ec2:DescribeTags           | View resource tags   |

# Best Practices

* Follow the Principle of Least Privilege
* Grant only required permissions
* Use IAM Groups for permission management
* Avoid wildcard permissions when possible
* Audit IAM Policies regularly
