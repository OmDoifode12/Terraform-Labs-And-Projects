# Day 30 – Delete EC2 Instance Using Terraform

---

## Task Overview

As part of AWS infrastructure cleanup and optimization, the Nautilus DevOps team needs to temporarily remove an unused EC2 instance using Terraform. The provisioning code must remain intact so the instance can be recreated later if required.

For this task:

* Delete the EC2 instance named:

```text id="dei01"
devops-ec2
```

using Terraform.

* Keep the Terraform provisioning code unchanged for future reuse.

* Ensure the EC2 instance reaches:

```text id="dei02"
terminated
```

state before submitting the task.

---

# Objective

| Property          | Value           |
| ----------------- | --------------- |
| Resource Type     | EC2 Instance    |
| Instance Name     | `devops-ec2`    |
| Action            | Delete Instance |
| Keep Code         | Yes             |
| AWS Region        | `us-east-1`     |
| Provisioning Tool | Terraform       |

---

# Understanding the Requirement

The important part of this task is:

```text id="dei03"
Delete infrastructure WITHOUT deleting Terraform code
```

This allows future reprovisioning.

---

# Existing Terraform Configuration

```hcl id="dei04"
resource "aws_instance" "ec2" {
  ami           = "ami-0c101f26f147fa7fd"
  instance_type = "t2.micro"

  vpc_security_group_ids = [
    "sg-8b11c48e58684e20b"
  ]

  tags = {
    Name = "devops-ec2"
  }
}
```

---

# Why Use Targeted Destroy?

Normally:

```bash id="dei05"
terraform destroy
```

removes ALL infrastructure.

But here we only want to delete:

```text id="dei06"
One EC2 instance
```

Terraform supports this using:

```bash id="dei07"
-target
```

---

# Architecture Overview

```text id="dei08"
Terraform State → Targeted Destroy → EC2 Instance Terminated
```

---

# Step-by-Step Implementation

## Step 1: Navigate to Terraform Directory

```bash id="dei09"
cd /home/bob/terraform
```

### Explanation

This is the Terraform working directory containing Terraform configuration files.

---

## Step 2: Verify Existing EC2 Instance

```bash id="dei10"
aws ec2 describe-instances \
--filters "Name=tag:Name,Values=devops-ec2" \
--query "Reservations[*].Instances[*].[InstanceId,State.Name]" \
--output table
```

### Explanation

Displays the EC2 instance status before deletion.

Expected State:

```text id="dei11"
running
```

---

## Step 3: Initialize Terraform

```bash id="dei12"
terraform init
```

### Explanation

Initializes Terraform and required providers.

---

## Step 4: Preview Targeted Destroy

```bash id="dei13"
terraform plan -destroy -target=aws_instance.ec2
```

### Explanation

Shows Terraform will delete only the EC2 instance.

Expected Output:

```text id="dei14"
Plan: 0 to add, 0 to change, 1 to destroy
```

---

## Step 5: Delete EC2 Instance

```bash id="dei15"
terraform destroy -target=aws_instance.ec2 -auto-approve
```

### Explanation

Deletes only the EC2 instance while keeping:

* Terraform code
* Terraform files
* Infrastructure configuration

Expected Output:

```text id="dei16"
Destroy complete! Resources: 1 destroyed.
```

---

# IMPORTANT

Do NOT delete:

```text id="dei17"
main.tf
```

The provisioning code must remain available for future deployments.

---

## Step 6: Verify EC2 Instance State

```bash id="dei18"
aws ec2 describe-instances \
--filters "Name=tag:Name,Values=devops-ec2" \
--query "Reservations[*].Instances[*].[InstanceId,State.Name]" \
--output table
```

### Explanation

Verify the instance state becomes:

```text id="dei19"
terminated
```

---

## Step 7: Verify Terraform State

Run:

```bash id="dei20"
terraform state list
```

### Explanation

The EC2 instance resource should no longer exist in Terraform state.

---

# Understanding Terraform State

Terraform tracks infrastructure resources inside:

```text id="dei21"
terraform.tfstate
```

After deletion:

```text id="dei22"
Resource removed from state
```

but:

```text id="dei23"
Terraform code remains intact
```

---

# Understanding Targeted Destroy

```bash id="dei24"
terraform destroy -target=RESOURCE
```

Deletes only the specified resource.

Useful for:

* Partial cleanup
* Testing
* Resource recreation
* Emergency removal

# Best Practices

* Always verify before deleting resources
* Use targeted destroy carefully
* Keep Terraform code version-controlled
* Avoid deleting production resources accidentally
* Verify infrastructure state after deletion

---

# Key Learnings

* Terraform can destroy specific resources selectively
* Terraform code can remain after resource deletion
* Targeted destroy enables controlled cleanup
* Terraform state tracks infrastructure resources
* Infrastructure as Code simplifies lifecycle management

# Interview Questions & Answers

### 1. What is Terraform destroy?

Terraform destroy removes managed infrastructure resources.

---

### 2. What does `-target` do in Terraform?

It limits operations to a specific resource.

---

### 3. Why use targeted destroy?

To delete only selected infrastructure resources.

---

### 4. Does Terraform delete code files during destroy?

No, Terraform only removes infrastructure resources.

---

### 5. Which Terraform command deletes a specific resource?

```bash id="dei30"
terraform destroy -target=RESOURCE_NAME
```

---

### 6. What is Terraform state?

Terraform state tracks managed infrastructure resources.

---

### 7. What command shows Terraform-managed resources?

```bash id="dei31"
terraform state list
```

---

### 8. What happens to an EC2 instance after deletion?

Its state becomes:

```text id="dei32"
terminated
```

---

### 9. Why keep Terraform code after deletion?

To recreate infrastructure later if needed.

---

* Automation
* Consistency
* Repeatability
* Version Control
* Reduced Human Error
