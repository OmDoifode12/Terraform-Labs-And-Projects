# Understanding Amazon SNS

Amazon SNS (Simple Notification Service) is a fully managed publish/subscribe messaging service provided by AWS.

SNS allows applications, services, and users to send and receive notifications instantly.

Common Use Cases:

* Application alerts
* Infrastructure notifications
* Monitoring systems
* Event-driven architectures
* Email notifications
* SMS alerts

---

# Understanding SNS Topics

An SNS Topic acts as a communication channel where messages are published.

Subscribers receive notifications from the topic.

Example Architecture:

```text
Applications → SNS Topic → Subscribers
```

Subscribers can include:

| Subscriber Type  |
| ---------------- |
| Email            |
| SMS              |
| Lambda Functions |
| SQS Queues       |
| HTTP Endpoints   |

# Step-by-Step Implementation

## Step 1: Navigate to Terraform Directory

```bash
cd /home/bob/terraform
```

### Explanation

This is the Terraform working directory where infrastructure configuration files are stored.

---

## Step 2: Create main.tf

```bash
touch main.tf
```

### Explanation

Terraform resources are written inside `.tf` files.

---

## Step 3: Open main.tf

```bash
vi main.tf
```

Press:

```text
i
```

to enter INSERT mode.

---

## Step 4: Add Terraform Configuration

```hcl
resource "aws_sns_topic" "devops_notifications" {
  name = "devops-notifications"
}
```

---

# Code Explanation

| Block         | Purpose               |
| ------------- | --------------------- |
| aws_sns_topic | Creates SNS Topic     |
| name          | SNS Topic name in AWS |

---

# Understanding SNS Topics

The SNS topic acts as a centralized notification channel.

Example:

```text
CloudWatch Alarm → SNS Topic → Email Notification
```

This enables real-time event-driven communication.

---

## Step 5: Save File

Press:

```text
ESC
```

Then type:

```text
:wq
```

and press ENTER.

---

## Step 6: Initialize Terraform

```bash
terraform init
```

### Explanation

Initializes Terraform and downloads required providers.

---

## Step 7: Validate Configuration

```bash
terraform validate
```

### Explanation

Checks Terraform syntax and configuration validity.

Expected Output:

```text
Success! The configuration is valid.
```

---

## Step 8: Preview Changes

```bash
terraform plan
```

### Explanation

Shows infrastructure changes before deployment.

---

## Step 9: Apply Configuration

```bash
terraform apply -auto-approve
```

### Explanation

Creates the SNS topic in AWS.

Expected Output:

```text
Apply complete! Resources: 1 added.
```

---

## Step 10: Verify SNS Topic

```bash
aws sns list-topics
```

### Explanation

Lists all SNS topics available in AWS.

Verify:

```text
devops-notifications
```

exists successfully.

---

## Step 11: Verify Topic Details

```bash
aws sns get-topic-attributes \
--topic-arn <TOPIC_ARN>
```

### Explanation

Displays SNS topic details including:

* Topic ARN
* Owner
* Display Name
* Access Policies

---

Before submitting the task, run:

```bash
terraform plan
```

Expected Output:

```text

```

This confirms Terraform state and AWS infrastructure are synchronized correctly.
