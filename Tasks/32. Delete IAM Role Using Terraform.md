# Day 32 – Delete IAM Role Using Terraform

---

## Task Overview

As part of AWS infrastructure cleanup and optimization, the Nautilus DevOps team needs to remove an unused IAM Role using Terraform. However, the Terraform provisioning code must remain intact so the role can be recreated later if required.

For this task:

* Delete the IAM Role named:

```text id="dir01"
iamrole_ravi
```

using Terraform.

* Keep the Terraform provisioning code unchanged.

---

# Objective

| Property          | Value          |
| ----------------- | -------------- |
| Resource Type     | IAM Role       |
| Role Name         | `iamrole_ravi` |
| Action            | Delete Role    |
| Keep Code         | Yes            |
| AWS Service       | IAM            |
| Provisioning Tool | Terraform      |

---

# Understanding IAM Roles

IAM Roles are AWS identities with temporary permissions.

Unlike IAM users:

```text id="dir02"
Roles are assumed temporarily
```

Common use cases:

* EC2 instance permissions
* Lambda execution roles
* Cross-account access
* Temporary credentials

---

# Why Delete IAM Roles?

Unused IAM roles can lead to:

* Security risks
* Permission sprawl
* Excessive access
* Management complexity

Cleanup improves:

* Security posture
* Access governance
* Infrastructure hygiene

# Step-by-Step Implementation

## Step 1: Navigate to Terraform Directory

```bash id="dir08"
cd /home/bob/terraform
```

### Explanation

This is the Terraform working directory containing Terraform configuration files.

---

## Step 2: Verify Existing IAM Role

```bash id="dir09"
aws iam get-role \
--role-name iamrole_ravi
```

### Explanation

Checks whether the IAM role exists before deletion.

Expected Output should contain:

```text id="dir10"
iamrole_ravi
```

---

## Step 3: Initialize Terraform

```bash id="dir11"
terraform init
```

### Explanation

Initializes Terraform and required providers.

---

## Step 4: Verify Terraform State

```bash id="dir12"
terraform state list
```

### Explanation

Confirms the IAM role resource exists in Terraform state.

Expected Output:

```text id="dir13"
aws_iam_role.role
```

---

## Step 5: Preview Targeted Destroy

```bash id="dir14"
terraform plan -destroy -target=aws_iam_role.role
```

### Explanation

Displays the Terraform plan for deleting only the IAM role.

Expected Output:

```text id="dir15"
Plan: 0 to add, 0 to change, 1 to destroy
```

---

## Step 6: Delete IAM Role

```bash id="dir16"
terraform destroy -target=aws_iam_role.role -auto-approve
```

### Explanation

Deletes only the IAM role while keeping:

* Terraform configuration
* main.tf
* Provisioning code

Expected Output:

```text id="dir17"
Destroy complete! Resources: 1 destroyed.
```

---

# IMPORTANT

Do NOT delete:

```text id="dir18"
main.tf
```

The provisioning code must remain available for future deployments.

---

## Step 7: Verify IAM Role Deletion

# Understanding Targeted Destroy

Terraform supports deleting individual resources using:

```bash id="dir22"
terraform destroy -target=RESOURCE_NAME
```

This enables:

* Partial cleanup
* Controlled deletion
* Safe infrastructure management

---

# Interview Questions & Answers

### 1. What is an IAM Role?

An IAM Role is an AWS identity with temporary permissions that can be assumed by trusted entities.

---

### 2. What is the difference between IAM User and IAM Role?

| IAM User              | IAM Role              |
| --------------------- | --------------------- |
| Permanent identity    | Temporary identity    |
| Long-term credentials | Temporary credentials |

---

### 3. What command deletes a specific Terraform resource?

```bash id="dir32"
terraform destroy -target=RESOURCE_NAME
```

---

### 4. Does Terraform destroy delete code files?

No, it only removes infrastructure resources.

### 5. What is Terraform state?

Terraform state tracks infrastructure managed by Terraform.

