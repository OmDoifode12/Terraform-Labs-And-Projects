# Day 08 – Create AMI Using Terraform

---

## Task Overview

As part of the AWS cloud migration initiative, the Nautilus DevOps team is automating infrastructure management using Terraform. To improve scalability, disaster recovery, and consistency across environments, the team needs to create an Amazon Machine Image (AMI) from an existing EC2 instance.

For this task, create an AMI named `nautilus-ec2-ami` from an existing EC2 instance named `nautilus-ec2` using Terraform.

---

# Objective

| Property          | Value              |
| ----------------- | ------------------ |
| Resource Type     | AMI                |
| Source Instance   | `nautilus-ec2`     |
| AMI Name          | `nautilus-ec2-ami` |
| Region            | `us-east-1`        |
| Provisioning Tool | Terraform          |

---

# Understanding AMI

An Amazon Machine Image (AMI) is a pre-configured template used to launch EC2 instances.

An AMI contains:

* Operating System
* Application Configuration
* Installed Software
* System Settings

Benefits:

* Faster server provisioning
* Consistent deployments
* Backup and recovery
* Golden image creation

# Step-by-Step Implementation

## Step 1: Navigate to Terraform Directory

```bash
cd /home/bob/terraform
```

### Explanation

This is the Terraform working directory where infrastructure configuration files are stored.

---

## Step 2: Identify Existing EC2 Instance

Run:

```bash
aws ec2 describe-instances \
--region us-east-1
```

### Explanation

This command lists all EC2 instances and helps identify the source instance used to create the AMI.

---

## Step 3: Create main.tf

```bash
touch main.tf
```

### Explanation

Terraform resources are defined inside `.tf` files.

---

## Step 4: Add Terraform Configuration

```hcl
data "aws_instances" "nautilus" {
  filter {
    name   = "tag:Name"
    values = ["nautilus-ec2"]
  }
}

resource "aws_ami_from_instance" "nautilus_ami" {
  name               = "nautilus-ec2-ami"
  source_instance_id = data.aws_instances.nautilus.ids[0]
}
```

### Explanation

| Parameter             | Description                     |
| --------------------- | ------------------------------- |
| aws_instances         | Retrieves existing EC2 instance |
| tag:Name              | Searches instance by Name tag   |
| aws_ami_from_instance | Creates AMI from EC2 instance   |
| name                  | AMI name                        |
| source_instance_id    | Source EC2 instance ID          |

---

## Step 5: Initialize Terraform

```bash
terraform init
```

### Explanation

Downloads required providers and initializes Terraform.

---

## Step 6: Validate Configuration

```bash
terraform validate
```

### Explanation

Checks Terraform syntax and configuration correctness.

---

## Step 7: Preview Changes

```bash
terraform plan
```

### Explanation

Displays the resources Terraform will create.

---

## Step 8: Apply Configuration

```bash
terraform apply -auto-approve
```

### Explanation

Creates the AMI from the specified EC2 instance.

---

## Step 9: Verify AMI

Run:

```bash
aws ec2 describe-images \
--owners self \
--region us-east-1
```

### Explanation

Lists available AMIs and verifies successful creation.

---

## Step 10: Wait for AMI Availability

Run:

```bash
aws ec2 describe-images \
--owners self \
--query "Images[*].[Name,State]" \
--output table
```

### Explanation

Ensure the AMI state becomes:

```text
available
```

before using it to launch new instances.

# Best Practices

* Use meaningful AMI names.
* Remove outdated AMIs regularly.
* Create AMIs before major changes.
* Use versioned AMI naming conventions.
* Test AMIs before production deployment.

---

# Key Learnings

* AMIs are reusable server templates.
* Terraform can automate AMI creation.
* AMIs improve consistency across environments.
* EC2 instances can be replicated easily using AMIs.
* Infrastructure backups can be automated using AMIs.

# Interview Questions & Answers

### 1. What is an AMI?

An Amazon Machine Image (AMI) is a template used to launch EC2 instances.

---

### 2. What does an AMI contain?

* Installed Software

---

### 3. Which Terraform resource creates an AMI from an EC2 instance?

---

### 4. Why are AMIs important?

They enable fast, consistent, and repeatable server deployments.

---

### 5. Can multiple EC2 instances be launched from the same AMI?

Yes, a single AMI can be used to launch multiple EC2 instances.

---

### 6. What is the difference between an AMI and a Snapshot?

| AMI                          | Snapshot                |
| ---------------------------- | ----------------------- |
| Complete machine template    | EBS volume backup       |
| Used to launch EC2 instances | Used to restore storage |
