# Day 31 – Delete IAM Group Using Terraform

---

## Task Overview

As part of AWS infrastructure cleanup and optimization, the Nautilus DevOps team needs to remove an unused IAM Group using Terraform. However, the Terraform provisioning code must remain intact so the group can be recreated later if needed.

For this task:

* Delete the IAM Group named:

```text id="dig01"
iamgroup_mark
```

using Terraform.

* Keep the Terraform provisioning code unchanged.

---

# Objective

| Property          | Value           |
| ----------------- | --------------- |
| Resource Type     | IAM Group       |
| Group Name        | `iamgroup_mark` |
| Action            | Delete Group    |
| Keep Code         | Yes             |
| AWS Service       | IAM             |
| Provisioning Tool | Terraform       |

---

# Understanding IAM Groups

IAM Groups are collections of IAM users.

Groups simplify permission management by allowing policies to be attached once and inherited by all users in the group.

Example:

```text id="dig02"
IAM Policy → IAM Group → Multiple Users
```

---

# Why Delete IAM Groups?

Unused IAM groups can create:

* Security risks
* Permission sprawl
* Management complexity

Regular cleanup improves:

* Security posture
* Access control hygiene
* Infrastructure management

# Step-by-Step Implementation

## Step 1: Navigate to Terraform Directory

```bash id="dig06"
cd /home/bob/terraform
```

### Explanation

This is the Terraform working directory containing Terraform configuration files.

---

## Step 2: Verify Existing IAM Group

```bash id="dig07"
aws iam get-group \
--group-name iamgroup_mark
```

### Explanation

Checks whether the IAM group exists before deletion.

Expected Output should contain:

```text id="dig08"
iamgroup_mark
```

---

## Step 3: Initialize Terraform

```bash id="dig09"
terraform init
```

### Explanation

Initializes Terraform and required providers.

---

## Step 4: Preview Targeted Destroy

```bash id="dig10"
terraform plan -destroy -target=aws_iam_group.this
```

### Explanation

Displays the Terraform plan for deleting only the IAM group.

Expected Output:

```text id="dig11"
Plan: 0 to add, 0 to change, 1 to destroy
```

---

## Step 5: Delete IAM Group

```bash id="dig12"
terraform destroy -target=aws_iam_group.this -auto-approve
```

### Explanation

Deletes only the IAM group while keeping:

* Terraform files
* Terraform configuration
* main.tf

Expected Output:

```text id="dig13"
Destroy complete! Resources: 1 destroyed.
```

---

# IMPORTANT

Do NOT delete:

```text id="dig14"
main.tf
```

The provisioning code must remain available for future use.

---

## Step 6: Verify IAM Group Deletion

```bash id="dig15"
aws iam get-group \
--group-name iamgroup_mark
```

### Explanation

Verifies the IAM group no longer exists.

Expected Output:

```text id="dig16"
NoSuchEntity
```

---

## Step 7: Verify Terraform State

```bash id="dig17"
terraform state list
```

### Explanation

The IAM group resource should no longer exist in Terraform state.

---

# Understanding Targeted Destroy

Terraform supports deleting individual resources using:

```bash id="dig18"
terraform destroy -target=RESOURCE_NAME
```

This allows:

* Partial cleanup
* Selective deletion
* Controlled infrastructure management

---

# Understanding Terraform State

Terraform tracks infrastructure resources inside:

```text id="dig19"
terraform.tfstate
```

After deletion:

```text id="dig20"
Resource removed from Terraform state
```

but:

```text id="dig21"
Terraform code remains unchanged
```

* Use targeted destroy carefully
* Verify resource deletion after cleanup
