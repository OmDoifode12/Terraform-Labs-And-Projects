# Day 21 – CloudWatch Setup Using Terraform

---

## Task Overview

As part of infrastructure monitoring and observability automation, the Nautilus DevOps team needs to configure Amazon CloudWatch resources using Terraform. CloudWatch helps monitor AWS resources, collect logs, and track application performance in real time.

For this task, create:

* A CloudWatch Log Group named `nautilus-log-group`
* A CloudWatch Log Stream named `nautilus-log-stream`

using Terraform.

# Understanding Amazon CloudWatch

Amazon CloudWatch is AWS’s monitoring and observability service.

It provides:

* Metrics collection
* Log monitoring
* Alerts and alarms
* Dashboards
* Application monitoring

CloudWatch helps teams monitor infrastructure health and troubleshoot issues efficiently.

| Security monitoring   |

# Understanding Log Groups

A Log Group is a container for log streams.

Example:

```text id="cwlog01"
Log Group → Multiple Log Streams
```

Log groups help organize logs by:

* Application
* Environment
* Service
* Resource type

---

# Understanding Log Streams

A Log Stream represents a sequence of log events from a single source.

Example Sources:

| Source Type  |
| ------------ |
| EC2 Instance |
| Lambda       |
| ECS Task     |
| Applications |

# Step-by-Step Implementation

## Step 1: Navigate to Terraform Directory

```bash id="cwlog03"
cd /home/bob/terraform
```

### Explanation

This is the Terraform working directory where infrastructure configuration files are stored.

---

## Step 2: Create main.tf

```bash id="cwlog04"
touch main.tf
```

### Explanation

Terraform resources are defined inside `.tf` files.

---

## Step 3: Open main.tf

```bash id="cwlog05"
vi main.tf
```

Press:

```text id="cwlog06"
i
```

to enter INSERT mode.

---

## Step 4: Add Terraform Configuration

```hcl id="cwlog07"
resource "aws_cloudwatch_log_group" "nautilus_group" {
  name = "nautilus-log-group"
}

resource "aws_cloudwatch_log_stream" "nautilus_stream" {
  name           = "nautilus-log-stream"
  log_group_name = aws_cloudwatch_log_group.nautilus_group.name
}
```

---

# Code Explanation

| Block                     | Purpose                      |
| ------------------------- | ---------------------------- |
| aws_cloudwatch_log_group  | Creates log group            |
| name                      | Log group name               |
| aws_cloudwatch_log_stream | Creates log stream           |
| log_group_name            | Associates stream with group |

---

# Understanding Resource Dependency

```hcl id="cwlog08"
log_group_name = aws_cloudwatch_log_group.nautilus_group.name
```

This automatically ensures:

```text id="cwlog09"
Log Group created first
```

before creating the Log Stream.

Terraform manages dependencies automatically.
