# Day 40 – Policy Variable Setup Using Terraform

---

## Task Overview

As part of AWS Identity and Access Management (IAM) automation, the Nautilus DevOps team needs to create an IAM Policy using Terraform while implementing variables for reusable and modular configurations.

For this task:

* Create an AWS IAM Policy.
* Store the IAM Policy name:

```text id="pvs01"
iampolicy_kirsty
```

inside a Terraform variable named:

```text id="pvs02"
KKE_iampolicy
```

---

# Objective

| Property          | Value              |
| ----------------- | ------------------ |
| Resource Type     | IAM Policy         |
| IAM Policy Name   | `iampolicy_kirsty` |
| Variable Name     | `KKE_iampolicy`    |
| AWS Service       | IAM                |
| AWS Region        | `us-east-1`        |
| Provisioning Tool | Terraform          |

---

# Understanding IAM Policies

IAM Policies define:

```text id="pvs03"
Permissions in AWS
```

Policies control:

* Allowed actions
* AWS resources
* Access levels

IAM Policies follow:

```text id="pvs04"
JSON format
```

---

# Why Use Variables in Terraform?

Terraform variables improve:

| Benefit     | Description                   |
| ----------- | ----------------------------- |
| Reusability | Reuse infrastructure easily   |
| Flexibility | Dynamic configuration values  |
| Modularity  | Cleaner Terraform code        |
| Scalability | Easier environment management |

Instead of hardcoding:

```hcl id="pvs05"
name = "iampolicy_kirsty"
```

we use:

```hcl id="pvs06"
name = var.KKE_iampolicy
```

# Configuring main.tf

```bash id="pvs15"

```

---

## Step 7: Add Terraform Configuration

```hcl id="pvs17"
resource "aws_iam_policy" "iam_policy" {
  name        = var.KKE_iampolicy
  description = "IAM Policy created using Terraform"

  policy = jsonencode({
    Version = "2012-10-17"

    Statement = [
      {
        Effect = "Allow"

        Action = [
          "ec2:Describe*"
        ]

        Resource = "*"
      }
    ]
  })

  tags = {
    Name = var.KKE_iampolicy
  }
}
```

---

# Code Explanation

| Block             | Purpose                        |
| ----------------- | ------------------------------ |
| aws_iam_policy    | Creates IAM Policy             |
| name              | IAM Policy name                |
| description       | Policy description             |
| policy            | IAM permission document        |
| jsonencode()      | Converts Terraform map to JSON |
| tags              | Adds AWS resource tags         |
| var.KKE_iampolicy | References Terraform variable  |

---

# Understanding IAM Policy Structure

An IAM Policy contains:

| Component | Purpose                 |
| --------- | ----------------------- |
| Version   | Policy language version |
| Statement | Permission block        |
| Effect    | Allow or Deny           |
| Action    | AWS operations          |
| Resource  | AWS resources           |

---

# Understanding jsonencode()

```hcl id="pvs18"
jsonencode()
```

converts Terraform data structures into valid JSON required by AWS IAM.

---

# Understanding Variable Reference

```hcl id="pvs19"
var.KKE_iampolicy
```

Terraform reads the value from:

```text id="pvs20"

```

and assigns:

```text id="pvs21"
iampolicy_kirsty
```

as the IAM Policy name.

---

# Understanding EC2 Describe Permissions

```hcl id="pvs22"
"ec2:Describe*"
```

allows:

* Viewing EC2 instances
* Viewing AMIs
* Viewing snapshots
* Viewing EC2 resources

This is:

```text id="pvs23"
Read-only access
```
