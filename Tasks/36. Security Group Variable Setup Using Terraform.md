# Day 36 – Security Group Variable Setup Using Terraform

---

## Task Overview

As part of AWS infrastructure automation, the Nautilus DevOps team needs to provision a Security Group using Terraform while implementing variables for reusable and modular configurations.

For this task:

* Create an AWS Security Group.
* Store the Security Group name:

```text id="sgv01"
nautilus-sg
```

inside a Terraform variable named:

```text id="sgv02"
KKE_sg
```

---

# Objective

| Property            | Value          |
| ------------------- | -------------- |
| Resource Type       | Security Group |
| Security Group Name | `nautilus-sg`  |
| Variable Name       | `KKE_sg`       |
| AWS Region          | `us-east-1`    |
| Provisioning Tool   | Terraform      |

---

# Understanding AWS Security Groups

AWS Security Groups act as:

```text id="sgv03"
Virtual Firewalls
```

for AWS resources.

They control:

* Inbound traffic
* Outbound traffic
* Network access rules

---

# Why Use Variables in Terraform?

Terraform variables improve:

| Benefit     | Description                   |
| ----------- | ----------------------------- |
| Reusability | Reuse infrastructure easily   |
| Flexibility | Change values dynamically     |
| Modularity  | Cleaner code organization     |
| Scalability | Easier environment management |

Instead of hardcoding:

```hcl id="sgv04"
name = "nautilus-sg"
```

we use:

```hcl id="sgv05"
name = var.KKE_sg
```

---

# Architecture Overview

```text id="sgv06"
variables.tf → main.tf → Security Group
```

---

# Terraform File Structure

| File         | Purpose                    |
| ------------ | -------------------------- |
| variables.tf | Stores Terraform variables |
| main.tf      | Creates AWS resources      |

# Step-by-Step Implementation

## Step 1: Navigate to Terraform Directory

```bash id="sgv07"
cd /home/bob/terraform
```

### Explanation

This is the Terraform working directory containing Terraform configuration files.

---

## Step 2: Create Terraform Files

```bash id="sgv08"
touch main.tf variables.tf
```

### Explanation

Creates the required Terraform files.

---

# Configuring variables.tf

## Step 3: Open variables.tf

```bash id="sgv09"
vi variables.tf
```

Press:

```text id="sgv10"
i
```

to enter INSERT mode.

---

## Step 4: Add Variable Configuration

```hcl id="sgv11"
variable "KKE_sg" {
  default = "nautilus-sg"
}
```

---

# Variable Explanation

| Block    | Purpose                     |
| -------- | --------------------------- |
| variable | Declares Terraform variable |
| KKE_sg   | Variable name               |
| default  | Default variable value      |

---

## Step 5: Save variables.tf

Press:

```text id="sgv12"
ESC
```

Then type:

```text id="sgv13"
:wq
```

and press ENTER.

---

# Configuring main.tf

## Step 6: Open main.tf

```bash id="sgv14"
vi main.tf
```

Press:

```text id="sgv15"
i
```

to enter INSERT mode.

---

## Step 7: Add Terraform Configuration

```hcl id="sgv16"
resource "aws_security_group" "nautilus_sg" {
  name        = var.KKE_sg
  description = "Security Group created using Terraform"

  tags = {
    Name = var.KKE_sg
  }
}
```

---

# Code Explanation

| Block              | Purpose                       |
| ------------------ | ----------------------------- |
| aws_security_group | Creates Security Group        |
| name               | Security Group name           |
| description        | Security Group description    |
| tags               | Adds resource tags            |
| var.KKE_sg         | References Terraform variable |

---

# Understanding Variable Reference

```hcl id="sgv17"
var.KKE_sg
```

Terraform reads the value from:

```text id="sgv18"
variables.tf
```

and assigns:

```text id="sgv19"
nautilus-sg
```

as the Security Group name.

---

# Understanding Security Group Rules

By default:

| Traffic Type | Behavior |
| ------------ | -------- |
| Inbound      | Denied   |
| Outbound     | Allowed  |

Security Groups are:

```text id="sgv20"
Stateful
```

meaning return traffic is automatically allowed.

```bash id="sgv28"

```

## Step 13: Verify Security Group Creation

```bash id="sgv30"

--filters "Name=group-name,Values=nautilus-sg" \
--query "SecurityGroups[*].[GroupId,GroupName]" \
--output table
```

Verifies the Security Group was created successfully.

Expected Output should contain:

```text id="sgv31"
nautilus-sg
```

