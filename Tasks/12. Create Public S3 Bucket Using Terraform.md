# Understanding Amazon S3

Amazon S3 (Simple Storage Service) is AWS's object storage service used for storing and retrieving data from anywhere.

Common Use Cases:

* Static website hosting
* Backup storage
* Application assets
* Log storage
* Media files

Benefits:

* High durability
* Unlimited scalability
* Cost-effective storage
* Global accessibility

---

# Understanding ACL

ACL (Access Control List) defines who can access an S3 bucket or its objects.

Common ACL Types:

| ACL                | Description                         |
| ------------------ | ----------------------------------- |
| private            | Only bucket owner has access        |
| public-read        | Anyone can read objects             |
| public-read-write  | Anyone can read and write           |
| authenticated-read | Any authenticated AWS user can read |

For this task:

```text
public-read
```

allows public access to bucket objects.

## Step 2: Create main.tf

```bash
touch main.tf
```

### Explanation

Terraform resources are defined inside `.tf` files.

---

## Step 3: Add Terraform Configuration

```hcl
resource "aws_s3_bucket" "devops_bucket" {
  bucket = "devops-s3-14874"
}

resource "aws_s3_bucket_public_access_block" "devops_public_access" {
  bucket = aws_s3_bucket.devops_bucket.id

  block_public_acls       = false
  block_public_policy     = false
  ignore_public_acls      = false
  restrict_public_buckets = false
}

resource "aws_s3_bucket_acl" "devops_bucket_acl" {
  depends_on = [aws_s3_bucket_public_access_block.devops_public_access]

  bucket = aws_s3_bucket.devops_bucket.id
  acl    = "public-read"
}
```

### Explanation

| Resource                          | Purpose                             |
| --------------------------------- | ----------------------------------- |
| aws_s3_bucket                     | Creates the S3 bucket               |
| aws_s3_bucket_public_access_block | Disables public access restrictions |
| aws_s3_bucket_acl                 | Makes bucket publicly readable      |

## Step 7: Apply Configuration


---

## Step 8: Verify Bucket Creation

```bash

```

### Explanation

Lists all available S3 buckets and confirms successful creation.

---

## Step 9: Verify Bucket ACL

```bash
aws --endpoint-url=http://aws:4566 s3api get-bucket-acl \
--bucket devops-s3-14874
```

### Explanation

Look for:

```text
AllUsers
READ
```

This confirms the bucket is publicly accessible.

---

## Step 10: Verify Public Access Configuration

```bash

--bucket devops-s3-14874
```

### Explanation

Verify all public access block settings are disabled.

# Best Practices

* Avoid public buckets unless absolutely necessary.
* Use bucket policies for fine-grained access control.
* Enable versioning for critical data.
* Apply least-privilege principles.
