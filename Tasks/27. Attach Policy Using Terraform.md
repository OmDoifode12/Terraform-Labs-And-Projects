# Understanding IAM Policies

IAM Policies define permissions in AWS.

Policies determine:

* What actions are allowed
* Which AWS resources can be accessed
* What operations users can perform

Example:

```text id="iap01"
EC2 Read Access
```

allows viewing EC2 resources.

---

# Understanding IAM Policy Attachments

Policies can be attached to:

| Attachment Type |
| --------------- |
| IAM Users       |
| IAM Groups      |
| IAM Roles       |


# Complete Terraform Configuration

```hcl id="iap10"
resource "aws_iam_user" "user" {
  name = "iamuser_siva"

  tags = {
    Name = "iamuser_siva"
  }
}

resource "aws_iam_policy" "policy" {
  name        = "iampolicy_siva"
  description = "IAM policy allowing EC2 read actions for siva"

  policy = jsonencode({
    Version = "2012-10-17"
    Statement = [
      {
        Effect   = "Allow"
        Action   = ["ec2:Read*"]
        Resource = "*"
      }
    ]
  })
}

resource "aws_iam_user_policy_attachment" "attach_policy" {
  user       = aws_iam_user.user.name
  policy_arn = aws_iam_policy.policy.arn
}
```

---

# Code Explanation

| Block                          | Purpose                 |
| ------------------------------ | ----------------------- |
| aws_iam_user                   | Creates IAM user        |
| aws_iam_policy                 | Creates IAM policy      |
| aws_iam_user_policy_attachment | Attaches policy to user |
| user                           | IAM username            |
| policy_arn                     | Policy ARN reference    |

---

# Understanding policy_arn

```hcl id="iap11"
policy_arn = aws_iam_policy.policy.arn
```

This references the ARN of the created IAM policy.

Terraform automatically resolves the ARN during deployment.
