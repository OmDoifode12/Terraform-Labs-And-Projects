# Day 29 – Delete Backup from S3 Using Terraform

---

## Task Overview

As part of AWS infrastructure cleanup and optimization, the Nautilus DevOps team needs to back up existing S3 bucket data locally before deleting the bucket. Terraform will be used along with AWS CLI commands to automate the process.

For this task:

1. Copy the contents of the S3 bucket:

```text id="dbs301"
xfusion-bck-29511
```

to:

```text id="dbs302"
/opt/s3-backup/
```

2. Delete the S3 bucket after backup completion.

3. Use Terraform with AWS CLI commands to accomplish the task.

---

| Property          | Value                  |

| Existing Bucket   | `xfusion-bck-29511`    |
| Backup Location   | `/opt/s3-backup/`      |


---

## Step 4: Open main.tf

```bash id="dbs310"
vi main.tf
```

Press:

```text id="dbs311"
i
```

to enter INSERT mode.

---

## Step 5: Add Terraform Configuration

```hcl id="dbs312"
resource "null_resource" "backup_and_delete_s3" {

  provisioner "local-exec" {
    command = <<EOT
aws s3 cp s3://xfusion-bck-29511 /opt/s3-backup/ --recursive
aws s3 rb s3://xfusion-bck-29511 --force
EOT
  }
}
```

---

# Code Explanation

| Block         | Purpose                       |
| ------------- | ----------------------------- |
| null_resource | Executes local shell commands |
| local-exec    | Runs commands locally         |
| aws s3 cp     | Copies S3 objects locally     |
| --recursive   | Copies all bucket contents    |
| aws s3 rb     | Removes S3 bucket             |
| --force       | Deletes bucket objects first  |

---

# Understanding Recursive Copy

```bash id="dbs313"
aws s3 cp s3://xfusion-bck-29511 /opt/s3-backup/ --recursive
```

This copies:

```text id="dbs314"
ALL bucket objects
```

to the local backup directory.

---

# Understanding Bucket Deletion

```bash id="dbs315"
aws s3 rb s3://xfusion-bck-29511 --force
```

This:

1. Deletes all bucket objects
2. Deletes the bucket itself
