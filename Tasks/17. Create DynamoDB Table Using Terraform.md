# Day 17 – Create DynamoDB Table Using Terraform

---

## Task Overview

As part of the AWS cloud migration initiative, the Nautilus DevOps team is provisioning database infrastructure using Terraform. They need a scalable NoSQL database solution for storing user-related information in real time.

For this task, create a DynamoDB table named `datacenter-users` using Terraform. The table must use `datacenter_id` as the primary key and should use `PAY_PER_REQUEST` billing mode.

# Understanding DynamoDB

Amazon DynamoDB is a fully managed NoSQL database service provided by AWS.

It is designed for:

* High performance
* Low latency
* Massive scalability
* Real-time applications

Common Use Cases:

* User management systems
* Session storage
* Gaming leaderboards
* IoT applications
* Real-time analytics

---

# Understanding PAY_PER_REQUEST

DynamoDB offers two billing modes:

| Billing Mode    | Description                       |
| --------------- | --------------------------------- |
| PROVISIONED     | Manual capacity management        |
| PAY_PER_REQUEST | Automatic scaling and pay-per-use |

For this task:

```text id="w95qfh"
PAY_PER_REQUEST
```

means AWS automatically scales read/write capacity.

No manual throughput management is required.

---

# Understanding Primary Key

The table uses:

```text id="4a5kn6"
datacenter_id
```

Every record in the table must have a unique value for this key.

## Step 4: Add Terraform Configuration

```hcl id="jlwmax"
resource "aws_dynamodb_table" "datacenter_users" {
  name         = "datacenter-users"
  billing_mode = "PAY_PER_REQUEST"

  hash_key = "datacenter_id"

  attribute {
    name = "datacenter_id"
    type = "S"
  }
}
```

---

# Code Explanation

| Block              | Purpose                |
| ------------------ | ---------------------- |
| aws_dynamodb_table | Creates DynamoDB table |
| name               | Table name             |
| billing_mode       | Defines billing method |
| hash_key           | Defines partition key  |
| attribute          | Defines key attributes |
| type = "S"         | String data type       |
